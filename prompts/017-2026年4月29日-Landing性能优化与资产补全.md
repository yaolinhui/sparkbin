## 一、Server Components 重构

### 1.1 背景
当前 7 个 Section 组件（Hero/Stages/Features/Deployment/Pricing/CTA/Footer）均为 `'use client'`，但它们只是静态展示，无客户端状态。导致不必要的 hydration 负担。

### 1.2 方案
- `app/[lang]/page.tsx` 改为 async Server Component
- 使用 `getTranslations` 获取所有翻译数据
- 将翻译文本/对象作为 props 传给 Section 组件
- Section 组件移除 `'use client'` 和 `useTranslations`
- 保持以下组件为 Client Component（真正需要客户端状态）：
  - `NavBar`（scroll listener、mobile menu）
  - `ThemeSwitcher`（localStorage）
  - `LanguageSwitcher`（useRouter）
  - `ScrollReveal`（IntersectionObserver）
  - `TerminalBlock`（无状态但含复杂渲染逻辑，可保留）
- `ScrollReveal` 用法不变，包裹需要动画的元素

### 1.3 关键实现
```tsx
// app/[lang]/page.tsx
export default async function HomePage({ params }: { params: { lang: string } }) {
  setRequestLocale(params.lang);
  const heroT = await getTranslations('hero');
  const stagesT = await getTranslations('stages');
  // ... 其他 namespace
  return (
    <>
      <NavBar />
      <main>
        <HeroSection
          badge={heroT('badge')}
          title1={heroT('title1')}
          // ... 其他字段
        />
        {/* ... */}
      </main>
    </>
  );
}
```
