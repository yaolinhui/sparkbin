# 任务 #19：修复 admin 密码稳定性问题（uvicorn reload 缓存）

## 问题描述
管理员密码在 uvicorn reload 后反复失效，表现为 "用户名或密码错误"。根本原因是：
1. Python `.pyc` 字节码缓存导致 reload 时可能加载旧版 `auth.py`
2. `hash_password` 与 `verify_password` 算法版本不一致时（如旧代码无 SHA-256 预哈希回退），验证直接失败
3. `init_default_user` 只做"不存在则创建"，不做"存在但哈希不兼容则修复"

## 修改文件

### 1. `backend/app/auth.py`（核心修改）
**允许修改：**
- `verify_password`：添加空值防护、独立异常处理与日志记录
- `init_default_user`：添加一致性自检（`hash_password`/`verify_password` 互验）+ 已存在 admin 的哈希兼容性自动修复逻辑
- 模块顶部：添加 `import logging` 和 `logger = logging.getLogger(__name__)`

**禁止修改：**
- `hash_password` 的算法本身（保持 SHA-256 预哈希 + bcrypt）
- `_prehash_password` 的实现
- `create_access_token` / `create_refresh_token` 的有效期
- `validate_password_complexity` 的复杂度规则

### 2. `backend/app/main.py`（基础设施加固）
**允许修改：**
- 文件顶部添加 `import sys` 和 `sys.dont_write_bytecode = True`
- `init_database()` 添加 try/except 日志包装
- 添加 auth 模块加载信息日志（记录文件路径和 mtime）

**禁止修改：**
- `_ensure_sqlite_columns` 中的任何列名或 SQL
- `lifespan` 的 `yield` 结构
- CORS、安全头、路由注册等已有配置

### 3. `backend/start.py`（启动脚本）
**允许修改：**
- 添加 `os.environ.setdefault("PYTHONDONTWRITEBYTECODE", "1")`

### 4. 新增 `backend/tests/test_auth.py`（测试覆盖）
**允许新增：**
- 密码哈希/验证往返测试
- 空值边界测试
- 旧 bcrypt 格式向后兼容测试
- `init_default_user` 自测试
- 密码复杂度校验测试
- 速率限制测试模式测试
