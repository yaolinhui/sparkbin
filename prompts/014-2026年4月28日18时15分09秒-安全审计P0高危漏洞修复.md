# 任务 #25：安全审计 P0 高危漏洞修复

## 问题描述

经对 SparkBin 全栈项目进行系统性安全审计，发现 5 个 P0 级高危漏洞，必须在上线前修复。这些问题涉及权限越权、开放重定向、账号接管和代码缺陷导致的 500 错误。

### P0-1：AI 配置接口缺少管理员权限校验（越权访问）
- **位置**：`backend/app/routers/ai.py`
- **具体端点**：
  - `GET /ai/configs`（第 128-145 行）：任何登录用户可查看所有 AI 提供商配置状态
  - `PUT /ai/configs/{provider}`（第 148-191 行）：任何登录用户可修改任意提供商的 API Key、Base URL、模型，甚至写入恶意配置
  - `POST /ai/test/{provider}`（第 194-209 行）：任何登录用户可测试任意 AI 配置，消耗后端流量
- **根因**：三个端点均只使用 `Depends(get_current_user)`，未使用 `Depends(require_admin)`
- **修复**：三个端点统一替换为 `Depends(require_admin)`，并确保 `from ..auth import require_admin` 已导入

### P0-2：Stripe 支付结算会话存在开放重定向漏洞
- **位置**：`backend/app/routers/payments.py`
- **具体代码**：`create_checkout_session` 函数（第 26-104 行）
- **根因**：`request.success_url` 和 `request.cancel_url` 直接来自用户请求体，未验证域名，即被传入 `stripe.checkout.Session.create`
- **修复**：增加域名白名单校验函数，仅允许 `settings.frontend_url` 及其同源域名

### P0-3：OAuth 账号绑定存在账号接管风险（Account Takeover）
- **位置**：`backend/app/routers/auth.py`
- **具体代码**：
  - Google OAuth 回调（第 655-665 行）：`existing.oauth_provider = "google"; existing.oauth_id = google_id` 直接覆盖
  - GitHub OAuth 回调（第 799-808 行）：`existing.oauth_provider = "github"; existing.oauth_id = github_id` 直接覆盖
- **根因**：当 OAuth 登录发现该 email 已存在本地账号时，**无条件直接覆盖** `oauth_provider` 和 `oauth_id`，没有任何密码验证或用户确认
- **修复**：如果该邮箱已存在且已有密码或其他 OAuth 绑定，禁止自动覆盖，返回 409 Conflict 并提示用户先登录现有账号再手动绑定

### P0-4：`generate_validate_suggestions` 因缩进错误未定义 — 500 错误 + 信息泄露
- **位置**：`backend/app/services/ai_proxy.py`（第 593 行）
- **根因**：`async def generate_validate_suggestions(...)` 被错误地缩进在 `_generate_idea_suggestions_single` 方法内部（多缩进了 4 个空格）。因此 `AIProxyService` 类实际**没有**该方法。当用户调用 `/ai/validate-suggest` 时，后端执行 `ai_service.generate_validate_suggestions(...)` 会抛出 `AttributeError`
- **修复**：将 `generate_validate_suggestions` 的缩进减少一级，使其与 `_generate_idea_suggestions_single` 同级（从 8 空格改为 4 空格）

### P0-5：生成推广建议时未校验项目所有权 — 水平越权（IDOR）
- **位置**：`backend/app/routers/ai.py`（第 370-378 行）
- **根因**：`generate_promote_suggestions` 接收可选的 `request.project_id`，保存 `PromoteSuggestion` 时直接关联传入的 `project_id`，**未验证该项目是否属于当前用户**
- **修复**：保存前查询数据库验证 `project_id` 对应的项目存在且 `user_id == current_user.id`

```python
from urllib.parse import urlparse
from ..config import get_settings

def _is_allowed_redirect_url(url: str) -> bool:
    settings = get_settings()
    allowed_domains = {
        urlparse(settings.frontend_url).netloc,
        # 生产环境如需补充更多域名，可在此扩展
    }
    parsed = urlparse(url)
    return parsed.scheme in ("http", "https") and parsed.netloc in allowed_domains

@router.post("/create-checkout-session", response_model=CheckoutSessionResponse)
def create_checkout_session(...):
    if not _is_allowed_redirect_url(request.success_url):
        raise HTTPException(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail="success_url 域名不在白名单中"
        )
    if not _is_allowed_redirect_url(request.cancel_url):
        raise HTTPException(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail="cancel_url 域名不在白名单中"
        )
    ...
```
