# Prompt：LandingPage 宣传网页升级

> **任务序号**：015
> **创建时间**：2026年4月28日 18:30:00
> **目标**：将 `frontend/src/components/LandingPage.tsx` 从极简占位页升级为完整的产品宣传落地页（Landing Page）

---

## 一、任务背景

当前 `LandingPage.tsx` 是一个仅包含品牌标题、六阶段 Grid 和"进入系统"按钮的极简占位页。作为 SparkBin 的访客首屏，它无法有效传达产品价值、六阶段方法论和开源+商业双轨模式，亟需升级为专业的产品宣传落地页。

---

## 二、项目上下文

### 技术栈
- 前端：React 18 + Vite + TypeScript + Tailwind CSS
- 图标库：`lucide-react`
- 样式系统：Brutalist（粗野主义）设计系统，CSS 变量 + `data-theme` 属性支持深色/浅色双主题
- 状态管理：Zustand（本任务不涉及）
- 路由：react-router-dom 6.x，BrowserRouter

### 现有 LandingPage 定位
- 文件路径：`frontend/src/components/LandingPage.tsx`
- 使用位置：`frontend/src/App.tsx` 中，当用户未登录时渲染，点击"进入系统"触发 `onEnter` 回调打开 `LoginModal`
- Props 接口：`interface LandingPageProps { onEnter: () => void; }`

### Brutalist 设计规范
- 等宽字体（JetBrains Mono）
- 零圆角边框（Tailwind 配置已全局覆盖 `border-radius: 0 !important`）
- 高对比度边框
- 颜色通过 CSS 变量使用：
  - `--brutal-bg` / `--brutal-surface` / `--brutal-surface-hover`
  - `--brutal-border` / `--brutal-border-hover`
  - `--brutal-text` / `--brutal-text-secondary` / `--brutal-muted`
  - `--brutal-accent` / `--brutal-accent-hover`
  - `--brutal-success` / `--brutal-warning` / `--brutal-error`

## 三、实现方案

### 3.1 页面结构（单页滚动式）

```
├── 固定导航栏（Nav）
│   ├── Logo + SPARKBIN
│   ├── 锚点链接：流程 / 功能 / 定价 / 开源
│   └── GitHub 链接 + 进入系统按钮
├── Hero 区
│   ├── 左侧：大标题 + 副标题 + CTA 按钮组 + 信任标签
│   └── 右侧：终端风格演示块（模拟 sparkbin CLI 输出）
├── 六阶段流程（Stages Flow）
│   ├── 桌面端：横向卡片流，带箭头连接
│   └── 移动端：纵向堆叠卡片
├── 核心功能矩阵（Features Grid）
│   └── 3x2 卡片：AI 宠物、GO/NO-GO、多平台文案、变现模拟器、项目蓝图、GitHub 备份
├── 开源 vs Cloud 双轨对比（Deployment）
│   ├── Self-Hosted：MIT + 无限 AI + 数据自主
│   └── Cloud：零配置 + 自动更新 + 托管 AI
├── 定价（Pricing）
│   └── Free $0 / Pro $9 / Team $29 三列对比
├── CTA Banner（二次转化）
└── Footer（品牌 / 产品 / 资源 / 开源 四栏）
```

### 3.2 设计细节

- **滚动动画**：使用 `IntersectionObserver` 实现 `ScrollReveal` 子组件，元素进入视口时淡入上浮（`opacity-0 translate-y-6` → `opacity-100 translate-y-0`）
- **导航栏滚动效果**：滚动超过 40px 时显示底部边框 + 半透明背景
- **CTA 按钮**：主按钮使用 `bg-brutal-accent`，次按钮使用 `bg-brutal-surface` + 边框
- **终端演示块**：模拟 macOS 窗口标题栏（红/黄/绿三点）+ 等宽字体 CLI 输出，展示六阶段工作流
- **响应式断点**：`md:`（768px+）、`lg:`（1024px+）

### 3.3 数据常量

页面内定义以下常量数组：
- `STAGES`：六阶段数据（num, key, label, labelZh, desc, icon）
- `FEATURES`：六个核心功能（icon, title, desc）
- `PRICING_TIERS`：三档定价（name, price, period, desc, features[], cta, accent）

## 四、代码规范（强制遵守）

1. **导出方式**：命名导出 `export function LandingPage`，禁止默认导出
2. **类型定义**：优先 `interface` 而非 `type`
3. **错误处理**：本组件不涉及 API 调用，UI 事件直接处理
4. **新增依赖**：不新增任何依赖，仅使用已有的 `lucide-react`
