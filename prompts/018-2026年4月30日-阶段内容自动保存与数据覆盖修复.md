# 任务 #29：阶段内容自动保存与数据覆盖修复

## 问题描述

用户在项目各阶段（想法/验证/原型/发布/增长/变现）中填写的内容，如果没有点击"提交此阶段"，在切换阶段或重新进入后内容会丢失。更严重的是，**多个阶段组件在初始化时会自动创建默认值并覆盖后端已有数据**，导致用户已保存的真实数据被静默覆盖。

经过全面排查，发现了以下三类缺陷：

### A类：数据覆盖（最严重）

**根因**：`ProjectDetail.tsx` 加载项目详情是异步的。子阶段组件在数据尚未加载完成时就已经渲染，此时 `project.stages?.[stage]?.content` 为空。多个阶段组件检测到 content 为空后，会自动创建默认值并调用 `onUpdateContent` 保存到后端，覆盖用户的真实数据。

**受影响组件**：
- `IdeaStage.tsx`：创建 `defaultNotes` 并自动 `saveNotes(defaultNotes)`
- `ValidateStage.tsx`：创建 `defaultItems` 并自动 `saveData(defaultData)`
- `PrototypeStage.tsx`：从 idea 阶段导入默认功能并自动 `saveData(initialData)`

### B类：高频保存无防抖

**根因**：`ProjectDetail.tsx` 的 `handleContentChange` 直接调用 `updateStageContent`，没有任何防抖。`RichTextEditor`（Tiptap）每次编辑操作都触发 `onChange`，导致用户每输入一个字符就发送一个 API 请求。

**后果**：
- 高频并发请求造成 race condition，后发的响应可能先返回，导致内容被旧数据覆盖
- 网络拥塞，后端压力增大
- 用户体验卡顿

### C类：useEffect 依赖不当

**根因**：多个阶段组件使用 `[project.id]` 或 `[project.painPoint]` 作为 `useEffect` 依赖。当 `ProjectDetail` 完成数据 fetch 并更新 store 后，组件重新渲染，但如果依赖值（如 `id`、`painPoint`）没有变化，`useEffect` 不会重新运行，导致组件不显示从后端加载的最新数据。

**受影响组件**：`IdeaStage`（依赖 `[project.painPoint]`）、`ValidateStage`、`PrototypeStage`、`ShipStage`、`GrowStage`、`MonetizeStage`（均依赖 `[project.id]`）

## 修改文件

### 1. `frontend/src/components/ProjectDetail.tsx`（防抖自动保存 + 加载状态保护）

**允许修改：**
- `handleContentChange`：改为防抖调用
- 新增 `saveStatus` / `lastSavedAt` state 用于显示保存状态
- 渲染逻辑：数据未加载完成前不渲染阶段子组件

**具体修改：**

在组件顶部添加：
```typescript
import { useRef, useCallback } from 'react'; // 确保已有

// 防抖保存
const saveTimeoutRef = useRef<ReturnType<typeof setTimeout> | null>(null);
const [saveStatus, setSaveStatus] = useState<'idle' | 'saving' | 'saved' | 'error'>('idle');
```

替换 `handleContentChange`：
```typescript
const handleContentChange = useCallback((content: string) => {
  if (saveTimeoutRef.current) {
    clearTimeout(saveTimeoutRef.current);
  }
  setSaveStatus('saving');
  saveTimeoutRef.current = setTimeout(async () => {
    try {
      await updateStageContent(project.id, validCurrentStage, content);
      setSaveStatus('saved');
      // 2 秒后恢复 idle
      setTimeout(() => setSaveStatus('idle'), 2000);
    } catch {
      setSaveStatus('error');
    }
  }, 1000); // 1 秒防抖
}, [project.id, validCurrentStage, updateStageContent]);
```

在 cleanup 中清除定时器：
```typescript
useEffect(() => {
  return () => {
    if (saveTimeoutRef.current) {
      clearTimeout(saveTimeoutRef.current);
    }
  };
}, []);
```

在 JSX 中添加保存状态指示器（放在阶段标题栏附近）：
```tsx
{saveStatus !== 'idle' && (
  <span className={`text-xs font-mono ml-2 ${
    saveStatus === 'saving' ? 'text-brutal-warning' :
    saveStatus === 'saved' ? 'text-brutal-success' :
    'text-brutal-warning'
  }`}>
    {saveStatus === 'saving' && '保存中...'}
    {saveStatus === 'saved' && '已保存'}
    {saveStatus === 'error' && '保存失败'}
  </span>
)}
```

**加载状态保护：**
在 `isLoading` 返回之后、`!project` 判断之前，添加一个判断：如果 `project` 存在但 stages 尚未加载（`!project.stages || Object.keys(project.stages).length === 0`），仍然显示 loading，直到 `ProjectDetail` 的 fetch effect 完成并填充数据。

```typescript
if (isLoading || !project || !project.stages || Object.keys(project.stages).length === 0) {
  return (
    <div className="min-h-screen bg-brutal-bg flex items-center justify-center font-mono">
      <div className="text-center border border-brutal-border bg-brutal-surface p-8">
        <div className="w-8 h-8 border-2 border-brutal-accent border-t-transparent animate-spin mx-auto mb-4" />
        <p className="text-brutal-muted text-sm">{'>'} LOADING_PROJECT_DATA...</p>
      </div>
    </div>
  );
}
```

**禁止修改：**
- `updateStageContent` 的调用时机（只在 `handleContentChange` 中调用）
- `markDirty` 和 `hasUnsavedChanges` 的逻辑（保留以用于浏览器返回拦截）

## 关键注意事项

1. **useEffect 依赖选择**：统一使用 `project.stages?.[stage]?.content` 作为依赖，而不是 `project.id` 或 `project.painPoint`。这样当 store 更新（fetch 完成后）时，effect 会重新运行并加载最新数据。

2. **不自动保存默认值**：`IdeaStage`、`ValidateStage`、`PrototypeStage` 的初始化逻辑中，当 `content` 为空时只设置本地 state，绝不调用 `saveData` / `saveNotes`。默认值的创建必须需要用户手动触发（按钮点击）。

3. **防抖与 dirty 状态共存**：`handleContentChange` 的防抖不影响 `markDirty` 机制。`RichTextEditor` 的 `onDirtyChange` 仍会在内容变化时立即触发，用于拦截浏览器返回；只有后端保存才走防抖。

4. **MonetizeStage 的防抖保存保留**：`MonetizeStage` 已有的 `saveDataDebounced`（600ms）是良好实践，本次只修复其 `useEffect` 依赖。

5. **空状态 UX**：当阶段内容为空时，显示空状态提示 + "使用默认模板"按钮，让用户明确知道当前是空的，并自主选择是否填充默认值。

6. **向后兼容**：已有的项目数据（已保存到后端的内容）不应受到任何影响。修改只影响初始化逻辑和保存行为。
