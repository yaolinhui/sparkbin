# 009 — 登录系统安全加固（上线前致命漏洞修复）

## 任务边界
**范围**：前后端认证系统（LoginModal、App 路由守卫、auth API、JWT 管理、RBAC、Rate Limit）
**不碰**：支付系统、AI Chat、项目面板业务逻辑
**分支**：master（直接提交）

---

## 验收标准（Must Have）

### 1. 前端 LoginModal.tsx
- [ ] 删除 `Default: admin / admin` 提示文本
- [ ] 用户名输入框默认清空（不再预填 `admin`）
- [ ] 增加关闭按钮（X），未登录时关闭后显示未登录占位页（而非强制卡在登录框）
- [ ] 密码输入框增加可见性切换（眼睛图标）
- [ ] 加载状态使用统一的 Brutalist 风格（禁用默认 `Loader2` 的 rounded-full）

### 2. 后端环境安全
- [ ] `SECRET_KEY`、`ENCRYPTION_KEY` 若仍为默认值，应用启动时直接 `sys.exit(1)` 并打印错误
- [ ] 删除 `main.py` 启动时打印默认密码到 stdout 的代码
- [ ] `default_username` / `default_password` 若仍为 `admin/admin`，启动时打印 WARNING

### 3. 后端登录 Rate Limit
- [ ] `POST /auth/login` 同一个 IP 5 分钟内最多 5 次失败尝试
- [ ] 超过限制返回 HTTP 429 + `Retry-After` 头
- [ ] 使用内存存储（`dict` + `deque`），不引入 Redis（保持项目轻量）

### 4. 后端 RBAC 执行
- [ ] 所有 `/admin/*` 接口必须校验 `current_user.role == "admin"`
- [ ] 非 admin 返回 HTTP 403
- [ ] 新增 `require_admin()` 依赖函数，复用在 admin router

### 5. 前端角色校验真实化
- [ ] `isAdmin()` 不再读取 `localStorage`，改为从 `/auth/me` 缓存的真实 role
- [ ] App.tsx 初始化时获取 `/auth/me`，把 role 写入 React state（或全局 store）
- [ ] 401 时不直接 `window.location.href = '/login'`，而是触发重新登录弹层

---

## 技术规范（遵守 CLAUDE.md）
- 命名导出，不用默认导出
- 类型定义优先 interface 而非 type
- 错误处理使用 Result<T, E> 模式（后端 FastAPI 用 HTTPException，前端用 throw Error）
- 禁止在代码中写死 API keys / secrets
