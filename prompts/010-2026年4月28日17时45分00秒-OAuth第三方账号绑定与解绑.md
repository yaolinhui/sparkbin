# 任务 #20：OAuth 第三方账号绑定与解绑

## 问题描述
当前 OAuth 支持「用 Google/GitHub 登录」，且**已有 GitHub 仓库授权连接**（`GET /oauth/github/connect`）。但缺少通用的「已登录用户绑定第三方账号」能力：
1. 已登录用户无法在账户设置中主动绑定/更换 Google 或 GitHub 账号
2. 没有解绑功能（至少保留一种登录方式，防止锁死）
3. `/auth/me` 没有返回 OAuth 绑定状态，前端无法展示连接/未连接

## 现有代码基础
- `backend/app/routers/auth.py` 第 856-950 行已有 `GET /oauth/github/connect` 和 `GET /oauth/github/connect/callback`，用于 GitHub 仓库授权
- `User` 模型为单 OAuth 设计（`oauth_provider` + `oauth_id`），采用覆盖式绑定策略

## 修改文件

### 1. `backend/app/routers/auth.py`（后端 API 扩展）
**允许修改：**
- 新增 `GET /auth/oauth/{provider}/bind` — 要求已登录（`Depends(get_current_user)`），生成含 `user_id` 的 state，跳转授权页
- 新增 `GET /auth/oauth/{provider}/bind/callback` — 绑定回调，将 `oauth_provider` + `oauth_id` 写入当前用户
  - 兼容 Google 和 GitHub 两个 provider
  - 绑定成功后重定向到前端个人资料页（带 `?oauth_bind_success=1`）
- 新增 `POST /auth/oauth/unbind` — 解绑，接收 `{ provider: "google" | "github" }`
  - 解绑前检查：若用户没有 `password_hash` 且要解绑的是当前唯一的 OAuth 方式，返回 400「至少保留一种登录方式」
- 修改 `GET /auth/me` 返回体：添加 `oauth_provider: string | null` 和 `oauth_id: string | null`（复用现有字段）

**禁止修改：**
- 现有的 `/oauth/google`、`/oauth/github` 登录端点（保持完全兼容）
- 现有的 `/oauth/github/connect` 仓库授权端点（保持完全兼容）
- `User` 模型的字段定义（维持单 OAuth 覆盖式绑定，不扩展 schema）
- 密码哈希逻辑

## 测试方案
1. 登录后点击「绑定 GitHub」→ 授权成功 → `/auth/me` 返回 `oauth_provider` = "github"
2. 再绑定 Google → Google 覆盖 GitHub（当前一对一限制）
3. 解绑当前 OAuth → 若用户有密码则成功；若用户无密码且是最后一种登录方式，返回错误提示
4. 使用绑定的 OAuth 账号登录 → 成功进入同一账户
