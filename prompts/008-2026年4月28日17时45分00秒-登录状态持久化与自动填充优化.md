# 任务 #22：登录状态持久化与自动填充优化

## 问题描述
当前登录体验存在三个问题：
1. 登录框每次打开都清空用户名和密码，用户需要反复输入
2. 没有"记住我"选项，无法按用户意愿控制持久化行为
3. 前端 token 刷新逻辑在边缘场景下可能不稳定（如网络抖动、服务端重启）

## 修改文件

### 1. `frontend/src/components/LoginModal.tsx`（登录弹窗优化）
**允许修改：**
- 在密码输入框下方添加「记住我」复选框（`type="checkbox"`）
- 使用 `localStorage` 存储 `sparkbin_remembered_username`（仅用户名，不存密码）
- 组件挂载时读取 `sparkbin_remembered_username` 自动填充到用户名输入框
- 登录成功时：若勾选记住我，保存用户名；若取消勾选，清除保存的用户名
- 注册成功后同样遵循记住我逻辑

**禁止修改：**
- 三栏切换结构（login/register/forgot）
- 蜜罐字段逻辑
- 密码强度指示器
- OAuth 按钮区域

### 2. `frontend/src/services/api.ts`（Token 刷新加固）
**允许修改：**
- `refreshAccessToken()`：添加重试机制（最多 2 次，间隔 1 秒），使用 `setTimeout` 而非连续立即重试
- `refreshAccessToken()`：刷新失败时调用 `clearAuthToken()` 和 `stopTokenRefreshTimer()`，再触发 `onUnauthorizedCallback`
- `request()`：401 后的刷新重试逻辑添加「仅重试一次」标记，防止无限循环
- 添加 `clearRememberedUsername()` 导出函数（供登录页调用）

**禁止修改：**
- `API_BASE_URL` 的读取方式
- Token 的 localStorage key 名（`sparkbin_token` / `sparkbin_refresh_token`）
- `extractErrorMessage` 的解析逻辑

### 3. `frontend/src/App.tsx`（登录状态检查优化）
**允许修改：**
- `OAuthHandler` 组件逻辑保持不变
- 在 `useEffect` 初始化登录状态时，添加对 token 有效性的前置检查（若 localStorage 中 token 存在但格式异常，直接清理并视为未登录）
- `startTokenRefreshTimer()` 调用时机：在确认登录状态为 true 之后立即启动

**禁止修改：**
- `AppRoutes` 的路由结构
- `isLoggedIn` / `userRole` / `requirePasswordChange` 三个核心状态的语义
- OAuth 回调 URL 解析逻辑

## 测试方案
1. 勾选「记住我」后登录 → 关闭弹窗 → 重新打开登录弹窗，用户名自动填充
2. 取消勾选「记住我」后登录 → 关闭弹窗 → 重新打开，用户名为空
3. 在已登录状态下将后端服务停止 → 等待 token 过期 → 前端应正确跳转登录页，不报错
4. `npm run build` 无 TypeScript 编译错误
