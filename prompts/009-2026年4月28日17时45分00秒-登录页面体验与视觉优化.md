# 任务 #21：登录页面体验与视觉优化

## 问题描述
经代码核查，当前 `LoginModal.tsx` 已具备较好的视觉基础（错误提示带图标、加载状态带文案、Tab 有切换样式、输入框有聚焦环），但仍有两处体验缺口：
1. **缺少键盘快捷键支持**：用户无法按 `Enter` 提交表单，无法按 `Esc` 关闭弹窗
2. **浏览器自动填充兼容性不足**：用户名输入框的 `autoComplete="off"` 应改为 `"username"`，以便浏览器正确识别并建议填充

## 修改文件

### 1. `frontend/src/components/LoginModal.tsx`（键盘与自动填充优化）
**允许修改：**
- 添加全局键盘监听 `useEffect`：
  - `Enter` 在登录/注册/忘记密码表单中触发表单提交（当前表单已有 `onSubmit`，但需确保非按钮元素聚焦时也能响应）
  - `Esc` 关闭弹窗（调用 `onClose`，需防重复触发）
- 用户名输入框：将 `autoComplete="off"` 改为 `autoComplete="username"`
- 注册表单的用户名输入框：添加 `autoComplete="username"`
- 注册表单的邮箱输入框：添加 `autoComplete="email"`
- 注册表单的密码输入框：确认已有 `autoComplete="new-password"`（如没有则添加）
- 注册表单的确认密码输入框：添加 `autoComplete="new-password"`
- 忘记密码的邮箱输入框：添加 `autoComplete="email"`

**禁止修改：**
- 整体 Brutalist 视觉风格（零圆角、粗边框、JetBrains Mono）
- 三栏结构（login/register/forgot）
- 蜜罐字段的实现方式
- OAuth 按钮区域
- `LoginModalProps` 接口定义
- 已有视觉元素：错误提示样式、加载状态文案、Tab 切换样式、输入框聚焦环

## 测试方案
1. 在用户名输入框按 `Enter` → 触发表单提交
2. 按 `Esc` → 弹窗关闭
3. 浏览器自动填充 → 用户名/密码应能正确识别和建议填充
4. `npm run build` 无 TypeScript 编译错误

## 提交标准
- 手动测试通过上述 4 项
- `npm run build` 成功
- 无视觉回归（其他页面样式不受影响）
