# 任务：邮箱注册 + Google/GitHub OAuth 登录实现

## 背景
当前系统只有预置的 admin 单用户，无注册、无多用户、无第三方登录。需要建立完整的用户认证体系，支持国际用户（Google/GitHub）和邮箱注册。

## 技术方案

### 1. 数据库变更（Alembic 迁移）
User 表新增字段：
- `email` (String(255), unique, indexed, nullable) — 邮箱
- `email_verified` (Boolean, default=False) — 邮箱是否验证
- `oauth_provider` (String(20), nullable) — 第三方来源：google/github
- `oauth_id` (String(255), nullable, indexed) — 第三方平台用户 ID
- `avatar_url` (String(500), nullable) — 头像 URL

约束：
- `username` 改为 nullable（OAuth 用户可能无用户名）
- `password_hash` 改为 nullable（OAuth 用户无密码）
- email 或 username + oauth_provider 至少有一个非空

### 2. 后端 API 新增

| 端点 | 方法 | 说明 |
|------|------|------|
| `/auth/register` | POST | 邮箱+用户名+密码注册，发验证邮件 |
| `/auth/verify-email` | GET | 邮箱验证回调 `?token=xxx` |
| `/auth/forgot-password` | POST | 提交邮箱，发送重置邮件 |
| `/auth/reset-password` | POST | 提交 token + 新密码 |
| `/auth/oauth/google` | GET | 跳转 Google OAuth 授权页 |
| `/auth/oauth/google/callback` | GET | Google 回调，创建/登录用户 |
| `/auth/oauth/github` | GET | 跳转 GitHub OAuth 授权页 |
| `/auth/oauth/github/callback` | GET | GitHub 回调，创建/登录用户 |

现有 `/auth/login` 保持兼容，继续支持 username/password 登录。

### 3. 邮件服务
- 使用 **Resend**（免费 3000 封/月）
- 环境变量：`RESEND_API_KEY`
- 邮件模板：注册验证、密码重置（纯文本即可，保持 Brutalist 风格）
- 验证/重置 token 使用 JWT，过期时间：24 小时

### 4. OAuth 配置
- Google：需要 `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`
- GitHub：需要 `GITHUB_CLIENT_ID`, `GITHUB_CLIENT_SECRET`
- 回调地址：`http://localhost:5173/auth/oauth/{provider}/callback`（开发环境）
- 生产环境通过 `FRONTEND_URL` 环境变量控制

### 5. 前端改动

LoginModal 重构：
- Tab 切换：「登录」/「注册」/「找回密码」
- 登录页保留用户名/密码，新增 "Or continue with" 分割线
- Google/GitHub OAuth 按钮（使用对应品牌色）
- 注册页：用户名、邮箱、密码、确认密码
- 找回密码页：邮箱输入 + 发送按钮

新增页面/组件：
- `VerifyEmailPage` — 处理 `/verify-email?token=xxx`
- `ResetPasswordPage` — 处理 `/reset-password?token=xxx`
- 路由注册在 App.tsx

### 6. 安全要求
- 密码仍使用现有复杂度校验（8位+大小写+数字）
- 注册时检查用户名和邮箱唯一性
- OAuth 回调必须校验 state 参数防 CSRF
- 邮箱验证前，OAuth 用户可登录但部分功能受限（标记 email_verified）

## 需要改动的文件清单

后端：
- `backend/app/models.py` — User 模型加字段
- `backend/app/schemas.py` — 新增 RegisterRequest, ForgotPasswordRequest 等 schema
- `backend/app/routers/auth.py` — 新增注册、邮件、OAuth 端点
- `backend/app/auth.py` — JWT 验证邮箱 token、OAuth 用户创建逻辑
- `backend/app/config.py` — 新增 Resend/OAuth 环境变量
- `backend/app/database.py` 或新增 `backend/app/email.py` — 邮件发送服务
- `backend/alembic/versions/` — 新增迁移文件

前端：
- `frontend/src/components/LoginModal.tsx` — 重构为三页签弹窗
- `frontend/src/App.tsx` — 新增 verify-email、reset-password 路由
- `frontend/src/services/api.ts` — 新增注册、找回密码、OAuth API 方法
- 新增 `frontend/src/pages/VerifyEmailPage.tsx`
- 新增 `frontend/src/pages/ResetPasswordPage.tsx`

环境配置：
- `backend/.env.example` — 新增 Resend/OAuth 配置项

## 规范
- 使用命名导出，不用默认导出
- 类型定义优先 interface 而非 type
- 错误处理使用 Result<T, E> 模式
- API keys 使用 env 变量，禁止写死
- 禁止修改已存在的 migrations 文件（只能新增）

## 实现顺序
1. 数据库迁移（User 模型 + Alembic）
2. 后端邮件服务 + Resend 集成
3. 后端注册/验证/找回密码 API
4. 后端 Google/GitHub OAuth
5. 前端 LoginModal 重构
6. 前端 VerifyEmailPage + ResetPasswordPage
7. 环境变量配置文档
