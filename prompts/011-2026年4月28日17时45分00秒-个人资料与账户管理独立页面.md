# 任务 #23：个人资料与账户管理独立页面

## 问题描述
当前用户没有独立的个人资料页面，所有信息修改散落在各处：
1. 主题偏好在某处修改，宠物配置在另一处，密码修改又在登录弹窗
2. 没有统一入口查看账户信息（注册时间、订阅状态、配额使用）
3. 没有退出登录按钮的常驻位置（目前可能只在某个角落）

## 修改文件

### 1. `frontend/src/components/ProfilePage.tsx`（新增独立页面）
**允许修改：**
- 新建文件，实现 `/profile` 路由对应的个人资料页
- 页面分区：
  - **账户概览**：用户名、邮箱、注册时间、角色标签
  - **配额信息**：本月 AI 调用次数/上限、项目数量/上限（调用 `/auth/me` 的 `quota` 字段）
  - **偏好设置**：主题切换（dark/light）、首选 AI 模型下拉选择
  - **宠物配置**：类型、名称、性格、 verbosity（若后端接口存在则调用，否则本地展示）
  - **安全设置**：修改密码按钮（唤起修改密码弹窗）、退出登录按钮
  - **OAuth 绑定**：Google/GitHub 绑定/解绑卡片（与任务 #20 联动）
- 使用 Brutalist 风格：粗边框、零圆角、卡片式布局
- 移动端适配：单列堆叠

**禁止修改：**
- 不使用任何外部 UI 库（保持现有纯 Tailwind + Lucide 方案）

### 2. `frontend/src/App.tsx`（路由注册）
**允许修改：**
- 在 `AppRoutes` 中添加 `/profile` 路由，要求登录态（未登录跳转 `/login`）
- 在顶部导航栏或侧边栏添加「个人资料」入口链接

**禁止修改：**
- `OAuthHandler` 组件
- `isLoggedIn` 状态的初始化逻辑（只添加路由，不改状态机）

### 3. `frontend/src/services/api.ts`（API 补充）
**允许修改：**
- 若缺失则补充 `authApi.updateTheme(theme)`、`authApi.updatePreferredModel(provider)`、`authApi.updatePetConfig(config)`
- 补充 `authApi.logout()`（如尚未存在）

**禁止修改：**
- 现有的 request 拦截和 401 处理逻辑

### 4. `backend/app/routers/auth.py`（后端接口补充）
**允许修改：**
- 确保 `/auth/me` 返回所有必要字段（已有）
- 确保 `PUT /preferred-model`、`PUT /me/pet-config`、`GET /theme`、`PUT /theme` 等端点正常工作（已有）

**禁止修改：**
- 不新增后端接口（前端整合现有接口）

## 测试方案
1. 登录后点击导航栏「个人资料」→ 进入 `/profile`，显示正确用户信息
2. 切换主题 → 刷新页面后主题保持
3. 修改首选模型 → 刷新后保持
4. 点击退出登录 → 清除 token 并跳转首页
5. 未登录时直接访问 `/profile` → 重定向到登录页
6. `npm run build` 无 TypeScript 编译错误

## 提交标准
- 页面布局符合 Brutalist 风格
- 所有可交互元素功能正常
- `npm run build` 成功
- 移动端无明显错位
