# Prompt：SparkBin 独立宣传网站（Landing Site）

> **任务序号**：016
> **创建时间**：2026年4月28日
> **任务目标**：基于 Next.js 14+ App Router 创建独立的宣传网站项目 `landing/`，与主应用（`frontend/` Vite SPA）完全分离，支持 SSR、多语言（zh/en）、SEO 优化，最终可独立部署。
> **技术方案**：方案 B — Next.js 独立项目

---

## 一、架构定位

| 维度 | 主应用（frontend/） | 宣传网站（landing/） |
|------|---------------------|----------------------|
| **框架** | Vite + React SPA | Next.js 14 App Router |
| **路由** | BrowserRouter（/project/:id, /admin） | 文件系统路由（/[lang]/） |
| **渲染** | 客户端渲染（CSR） | 服务端渲染（SSR）+ SSG |
| **用途** | 产品功能（需登录） | 产品宣传、获客、SEO |
| **部署** | app.sparkbin.dev（或子路径） | www.sparkbin.dev（根域名） |
| **代码共享** | 不共享组件（避免耦合） | 可复用设计规范（颜色、字体、Tailwind 配置） |

**核心原则**：宣传网站与主应用完全解耦，独立开发、独立构建、独立部署。

### 3.4 全局样式（globals.css）

复用主项目 `frontend/src/index.css` 中的 CSS 变量和工具类，适配 Next.js：

- `:root` 深色主题变量
- `[data-theme="light"]` 浅色主题变量
- `*` 全局 `border-radius: 0 !important`
- `body` dot grid background（radial-gradient）
- `.btn-brutal` / `.btn-brutal-primary` 工具类
- 全部 `bg-brutal-*` / `text-brutal-*` / `border-brutal-*` 工具类

**主题切换策略**：
1. 默认深色主题（CSS 变量 `:root`）
2. 客户端挂载后检测 `localStorage.getItem('theme')`，设置 `document.documentElement.setAttribute('data-theme', ...)`
3. 主题切换按钮仅客户端渲染（`'use client'`），避免 SSR hydration mismatch

## 四、设计语言系统（Design Language System）

### 4.1 设计原则（Design Principles）

1. **信息密度优先**：每屏必须传达明确价值，禁止大段无意义留白
2. **终端美学**：模仿 CLI / 终端的视觉语言，代码感、命令感、系统感
3. **结构即内容**：边框不是装饰，而是信息架构的一部分
4. **零装饰**：无阴影（或极弱）、无渐变、无圆角、无模糊背景
5. **动效有目的**：每个动画必须服务于引导注意力或反馈状态

### 4.2 视觉层级系统（Typography Scale）

| 层级 | 用途 | 桌面端 | 移动端 | 字重 | 行高 |
|------|------|--------|--------|------|------|
| Display | Hero 主标题 | 60px | 36px | Bold (700) | 1.1 |
| H1 | 区块标题 | 30px | 24px | Bold (700) | 1.2 |
| H2 | 卡片标题 | 18px | 16px | Bold (700) | 1.3 |
| H3 | 小标题 | 14px | 14px | SemiBold (600) | 1.4 |
| Body | 正文 | 14px | 14px | Regular (400) | 1.6 |
| Caption | 标签、辅助文字 | 12px | 12px | Regular (400) | 1.4 |
| Micro | 版权、元信息 | 10px | 10px | Regular (400) | 1.4 |

字体栈：`font-family: 'JetBrains Mono', 'Fira Code', 'SF Mono', 'Consolas', monospace;`

### 4.3 间距系统（Spacing Scale）

基于 4px 网格系统：

| Token | 值 | 用途 |
|-------|-----|------|
| xs | 4px | 图标与文字间隙 |
| sm | 8px | 紧凑内边距 |
| md | 16px | 标准内边距、列间距 |
| lg | 24px | 卡片内边距 |
| xl | 32px | 区块内部间距 |
| 2xl | 48px | 大区块间隙 |
| 3xl | 64px | Section 垂直内边距（移动端） |
| 4xl | 96px | Section 垂直内边距（桌面端） |

**容器规范**：
- 最大宽度：1152px (72rem)
- 水平内边距：桌面 32px，移动端 16px
- 列间距（gap）：16px (gap-4)

### 4.4 网格系统（Grid System）

| 布局 | 列数 | 断点 |
|------|------|------|
| Hero | 1 列（移动端）/ 2 列（桌面端） | md (768px) |
| Stages | 1 列（移动端）/ 6 列（桌面端） | lg (1024px) |
| Features | 1 列（移动端）/ 2 列（平板）/ 3 列（桌面端） | sm (640px) / lg |
| Deployment | 1 列（移动端）/ 2 列（桌面端） | md |
| Pricing | 1 列（移动端）/ 3 列（桌面端） | md |
| Footer | 1 列（移动端）/ 2 列（平板）/ 4 列（桌面端） | sm / md |

### 4.5 色彩使用规则（Color Usage Rules）

| 颜色变量 | 正确用法 | 禁止用法 |
|----------|----------|----------|
| `--brutal-accent` | 主 CTA 背景、强调文字、活跃状态边框、关键标签 | 大面积背景填充（超过 200x200px） |
| `--brutal-success` | 完成状态、积极反馈、GO 决策标记 | 作为文字颜色在浅色背景上使用 |
| `--brutal-warning` | 警告状态、需要注意的信息 | 与 `--brutal-error` 同时使用于同一元素 |
| `--brutal-error` | 错误状态、NO-GO 决策标记 | 作为装饰色或大面积使用 |
| `--brutal-text` | 主要文字、标题 | 深色背景上使用深色值 |
| `--brutal-text-secondary` | 次要文字、描述、正文 | 用于主要行动按钮文字 |
| `--brutal-muted` | 禁用状态、元信息、分隔线、占位符 | 用于正文内容（对比度不足） |
| `--brutal-border` | 卡片边框、分隔线、输入框边框 | 用于文字或图标颜色 |
| `--brutal-bg` | 页面背景、底层容器 | 用于前景元素背景（无法区分层级） |

### 4.6 组件设计规范（Component Design Tokens）

**按钮变体**：

| 变体 | 背景 | 文字 | 边框 | 悬停效果 | 用途 |
|------|------|------|------|----------|------|
| Primary | `--brutal-accent` | `--brutal-bg` | `--brutal-accent` | 反色（bg 变透明，文字变 accent） | 主 CTA |
| Secondary | `--brutal-surface` | `--brutal-text` | `--brutal-border` | 边框变 `--brutal-text` | 次 CTA |
| Ghost | transparent | `--brutal-muted` | transparent | 文字变 `--brutal-text` | 导航链接 |
| Accent Outline | transparent | `--brutal-accent` | `--brutal-accent` | 背景填充 accent，文字变 bg | 特殊强调 |

**按钮尺寸**：
- 大：px-6 py-3（Hero CTA、CTA Banner）
- 中：px-5 py-2.5（卡片内 CTA）
- 小：px-3 py-1.5（导航栏按钮）

**卡片规范**：
- 边框：2px solid（默认 `--brutal-border`，强调 `--brutal-accent`）
- 内边距：24px (p-6)
- 悬停：边框色过渡到 accent，背景过渡到 surface-hover，过渡时长 150ms
- 禁止：阴影、圆角、模糊效果

**输入框规范**：
- 背景：`--brutal-surface`
- 边框：1px solid `--brutal-border`
- 聚焦：边框色变为 `--brutal-accent`
- 占位符：`--brutal-muted`
- 内边距：px-3 py-2

**链接规范**：
- 默认：`--brutal-text-secondary`，无下划线
- 悬停：`--brutal-text`，下划线出现或颜色变化
- 外部链接：尾部带 `ExternalLink` 图标（12px）
- 过渡：75ms

**标签/徽章规范**：
- 边框：1px solid
- 内边距：px-2 py-0.5
- 字体：10px uppercase tracking-widest
- 强调标签：边框和文字使用 `--brutal-accent`

### 4.7 内容策略（Content Strategy）

**语气与语调（Voice & Tone）**：
- **Voice（品牌人格）**：专业、直接、略带技术傲慢（适合开发者群体）、不讨好、不解释显而易见的事
- **Tone（具体表达）**：
  - 中文用"你"不用"您"
  - 英文不用"we"而用直接陈述（"Validate before you build" 而非 "We help you validate"）
  - 禁止营销腔、禁止过度承诺

**文案规范**：
- 标题：英文全大写 + tracking-tight，中文正常
- 按钮文案：动词开头，不超过 4 个中文字符或 2 个英文单词
- 标签：英文全大写 + tracking-widest（10px）
- 正文： sentence case，禁止全大写长段落

**禁止词汇**：
- "最好的"、"领先的"、"革命性的"、"颠覆性的"、"独一无二的"
- "只需一键..."、"轻松..."、"简单..."
- **替代原则**：用事实代替形容词，用数据代替承诺

**CTA 文案优先级**：
1. 主 CTA："进入系统" / "Get Started" / "免费开始"
2. 次 CTA："在 GitHub 上查看" / "View on GitHub" / "Star on GitHub"
3. 辅助 CTA："自托管指南" / "Self-Hosting Guide" / "查看文档"

## 五、动效与交互规范（Motion & Interaction）

### 5.1 动画原则

1. **功能性优先**：每个动效必须服务于引导注意力或提供状态反馈
2. **克制**：单页同时运动的动画元素不超过 3 个
3. **可预测**：相同交互在不同位置产生相同结果
4. **尊重用户偏好**：支持 `prefers-reduced-motion`，此时禁用所有非必要动画

### 5.2 过渡标准

| 类型 | 时长 | 缓动函数 | 用途 |
|------|------|----------|------|
| 快速反馈 | 75ms | ease-out | 按钮悬停、边框变色、链接下划线 |
| 状态切换 | 150ms | ease-in-out | 主题切换、标签切换、卡片悬停 |
| 内容浮现 | 300-500ms | ease-out | ScrollReveal 淡入、模态框出现 |
| 页面过渡 | 300ms | ease-in-out | 语言切换（路由变化） |

### 5.3 滚动行为

- 锚点滚动：`scroll-behavior: smooth`
- Nav 点击锚点：平滑滚动到对应 section，偏移量考虑固定 Nav 高度（56px）
- 禁止：全局视差滚动（与 Brutalist 的直接感冲突）
- 返回顶部：滚动超过 500px 后显示返回顶部按钮（可选）

### 5.4 悬停状态（Hover States）

| 元素 | 悬停效果 | 时长 | 说明 |
|------|----------|------|------|
| Primary 按钮 | 背景/文字反色 | 75ms | bg 变透明，文字变 accent |
| Secondary 按钮 | 边框变 text 色 | 75ms | --brutal-border → --brutal-text |
| 卡片 | 边框变 accent | 150ms | 同时 bg 变 surface-hover |
| 链接 | 颜色加深 | 75ms | 或下划线出现 |
| 图标按钮 | 颜色变 accent | 75ms | --brutal-muted → --brutal-accent |

**禁止的悬停效果**：
- 放大（transform: scale）
- 阴影扩散（box-shadow 扩散）
- 模糊效果（filter: blur）
- 倾斜或旋转

### 5.5 焦点状态（Focus States）— 无障碍要求

- 键盘导航：`outline: 2px solid var(--brutal-accent); outline-offset: -2px`
- 鼠标点击不显示 focus ring：使用 `:focus-visible` 而非 `:focus`
- 所有可交互元素必须有可见焦点状态
- Tab 顺序符合视觉顺序（从左到右、从上到下）

### 5.6 加载状态

- 首屏：无需 loading spinner（SSR 直接输出 HTML）
- 异步内容：使用骨架屏或极简脉冲动画（`animate-pulse`）
- 禁止：旋转菊花（spinning loader）、进度条、弹窗 loading overlay

## 六、多语言方案（next-intl）

### 6.1 路由策略

- `/zh` — 中文（默认语言）
- `/en` — 英文
- `/` — 自动重定向到 `/zh`（或根据浏览器语言偏好）

### 6.2 实现方式

使用 `next-intl` + App Router 的 `[lang]` 动态路由：

```typescript
// app/[lang]/layout.tsx
import { NextIntlClientProvider } from 'next-intl';
import { getMessages } from 'next-intl/server';

export default async function LocaleLayout({
  children,
  params: { lang },
}: {
  children: React.ReactNode;
  params: { lang: string };
}) {
  const messages = await getMessages();
  return (
    <html lang={lang}>
      <body>
        <NextIntlClientProvider messages={messages} locale={lang}>
          {children}
        </NextIntlClientProvider>
      </body>
    </html>
  );
}
```

- **图片**：`loading="lazy"` 懒加载，WebP 格式优先，提供 fallback
- **第三方脚本**：`defer` 或 `async`，禁止阻塞渲染的脚本
- **代码分割**：按路由自动分割，首屏只加载必要 JS

## 十三、代码规范（硬规则）

1. **导出方式**：命名导出 `export function Xxx`，禁止默认导出
2. **类型定义**：优先 `interface` 而非 `type`
3. **客户端组件**：需要 `useEffect` / `useState` / `useRef` 的组件顶部必须标注 `'use client'`
4. **服务端组件**：默认服务端渲染，不标注 `'use client'`，SEO 相关逻辑优先放服务端
5. **样式**：Tailwind 工具类 + 自定义 CSS 变量，禁止内联 `style={{}}`（除动态计算值外）
6. **图标**：仅使用 `lucide-react`，不引入其他图标库
7. **图片**：使用标准 `<img>` 标签（静态导出无法使用 Next.js Image 组件优化），必须提供 `alt`
8. **新增依赖**：任何新依赖必须说明理由并检查 bundle size 影响
9. **构建验证**：`npm run build` 必须通过，零 TypeScript 错误
10. **翻译 key**：使用扁平结构（`nav.stages`），不使用深层嵌套（超过 3 层）

## 十七、品牌识别系统（Brand Identity System）

### 17.1 Logo 使用规范

**主 Logo 形式**：

- 图标尺寸比例：图标高度 = 文字 cap-height 的 1.2 倍
- 最小安全空间：Logo 四周留白 ≥ Logo 高度的 0.5 倍

**独立图标使用场景**：
- Favicon：16x16 / 32x32 / 180x180（apple-touch-icon）
- 社交媒体头像：400x400，中心放置 Lock 图标
- 导航栏：缩小版组合 Logo（图标 32px + 文字）

**禁止用法**：
- 拉伸或压缩 Logo 的宽高比
- 旋转 Logo（任何角度）
- 修改 Logo 颜色（除非在特定深色/浅色背景下使用反色版本）
- 为 Logo 添加阴影、描边、渐变
- 将 Lock 图标替换为其他图标

**反色版本**：
- 深色背景：白色图标 + 白色文字
- 浅色背景：黑色图标 + 黑色文字

### 19.2 技术扩展机制

**添加新语言的步骤**：

1. **翻译文件**：创建 `i18n/messages/[lang].json`，复制 `zh.json` 结构并翻译
2. **路由配置**：在 `i18n/config.ts` 的 `locales` 数组中添加新语言代码
3. **路由中间件**：`middleware.ts` 自动适配，无需修改（`[lang]` 动态路由）
4. **SEO**：在 `generateMetadata` 的 `alternates.languages` 中添加新语言映射
5. **导航**：`LanguageSwitcher` 组件添加新语言选项
6. **字体检查**：确认新语言字符集与 JetBrains Mono 兼容（CJK 已覆盖，阿拉伯语/希伯来语需额外处理）

**翻译文件管理**：
- 使用扁平 key 结构，最大深度 3 层
- 所有翻译文件 key 必须保持一致（en.json 有的 key，zh.json 必须也有）
- 缺失 key 回退到 `defaultLocale`（zh），控制台 warning

### 19.3 RTL（从右到左）支持评估

**当前状态**：不支持 RTL。

**影响语言**：阿拉伯语（ar）、希伯来语（he）、波斯语（fa）等。

**如需支持 RTL，需进行以下改造**：
- Tailwind：使用 `rtl:` 修饰符（如 `rtl:text-right`）
- CSS：使用逻辑属性（`margin-inline-start` 替代 `margin-left`）
- 布局：Flexbox 和 Grid 天然支持 RTL（`dir="rtl"` 自动翻转）
- 图标：箭头方向类图标需镜像翻转
- 评估工作量：+30% CSS 工作量，建议作为 P2 后续扩展

### 19.4 区域化差异（Localization）

**日期格式**：
- zh：`YYYY-MM-DD`（2026-04-28）
- en：`MMM DD, YYYY`（Apr 28, 2026）或 `DD MMM YYYY`（28 Apr 2026）
- ja：`YYYY年MM月DD日`

**数字格式**：
- 千分位分隔符：zh/en 用逗号 `1,000`；部分欧洲语言用空格或点
- 小数点：zh/en 用点 `9.99`；部分欧洲语言用逗号

**货币显示**：
- USD 符号位置：en 用前缀 `$9`，部分语言用后缀 `9 $`
- 当前影响较小（landing 页数字少），但 PricingSection 需支持区域化渲染

**地址/电话格式**：
- landing 页暂不提供联系表单，此项预留

## 二十一、安全规范（Security）

### 21.1 内容安全策略（CSP）

**HTTP 头配置**（静态导出时通过 CDN/Nginx 配置，或在 HTML meta 标签中设置）：

```
Content-Security-Policy:
  default-src 'self';
  script-src 'self' 'unsafe-inline';
  style-src 'self' 'unsafe-inline';
  img-src 'self' data: https:;
  font-src 'self';
  connect-src 'self' https://plausible.io;
  frame-src 'none';
  object-src 'none';
  base-uri 'self';
  form-action 'self';
```

**说明**：
- `'unsafe-inline'` 用于 Tailwind 的实用类内联样式（静态导出无法使用 nonce）
- 生产环境建议通过构建工具计算 style-src hash 替代 `'unsafe-inline'`
- 所有外部资源（分析、字体 CDN）必须在 CSP 中白名单化
- 禁止：`unsafe-eval`（无 eval / new Function 需求）

### 21.2 XSS 防护

- **输出编码**：React 默认转义 JSX 中的变量，禁止直接使用 `dangerouslySetInnerHTML`
- **外部链接**：所有 `<a href>` 到外部域名必须设置 `rel="noopener noreferrer"`
- **URL 验证**：任何用户输入的 URL（如社交分享参数）必须经过验证和清洗
- **富文本**：landing 页不提供用户输入的富文本展示，如未来需要必须使用 DOMPurify

### 21.3 点击劫持防护

- **X-Frame-Options**: `DENY`（禁止任何 iframe 嵌入）
- **CSP frame-ancestors**: `'none'`（同上，现代浏览器标准方式）
- 例外：如未来需要嵌入 ProductHunt 展示，可放宽为 `SAMEORIGIN`

### 21.4 HTTPS 与传输安全

- **强制 HTTPS**：所有资源（图片、CSS、JS）必须通过 HTTPS 加载
- **HSTS**（HTTP Strict Transport Security）：
  ```
  Strict-Transport-Security: max-age=31536000; includeSubDomains; preload
  ```
- **混合内容**：禁止 HTTP 资源嵌入 HTTPS 页面（浏览器默认阻止）

### 21.5 敏感信息保护

- **禁止硬编码**：API Key、密钥、密码等敏感信息禁止写入客户端代码
- **环境变量**：构建时通过 `.env` 注入，但不暴露给浏览器（Next.js 以 `NEXT_PUBLIC_` 前缀区分的变量会暴露）
- **Git 保护**：`.env*` 必须加入 `.gitignore`
- **审计**：定期运行 `npm audit` 检查依赖漏洞

### 21.6 依赖安全

- **漏洞检查**：每次添加依赖前运行 `npm audit`
- **过期依赖**：每月检查并更新过期依赖
- **最小权限**：不安装不需要的依赖（避免 supply chain 攻击面）
- **锁定文件**：使用 `package-lock.json` 锁定版本，防止意外升级
