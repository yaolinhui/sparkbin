# 任务 #16：开源许可证与社区文档完善

## 目标
为 SparkBin 选择并应用合适的开源许可证，完善社区文档，使项目达到可直接公开发布的标准。

## 现状
- `prompts/011-2026年4月27日-开源准备.md` 是早期简化版，仅要求 MIT + 英文 README
- 仓库目前无 `LICENSE` 文件
- `README.md` 是中文，缺少自托管说明、截图、徽章
- `CONTRIBUTING.md` 可能已存在但需更新
- 无 `SECURITY.md`
- 无 `CHANGELOG.md` 规范
- GitHub 相关模板缺失（Issue、PR 模板）

## 执行步骤

### 1. 选择并创建 LICENSE 文件
- 文件：根目录 `LICENSE`
- **推荐使用 Server Side Public License (SSPL) v1** 或 **Elastic License 2.0 (ELv2)**
  - 原因：允许个人/小公司自托管免费使用；防止云厂商（AWS/Azure）直接封装成 SaaS 售卖，保护未来的商业空间
  - 如果倾向更宽松：用 AGPL-3.0（网络交互触发开源义务）
  - 如果倾向最宽松：用 MIT（但无云保护）
- 文件中需包含：许可证全文、版权声明（`Copyright (c) 2026 [你的名字/组织名]`）

### 2. 重写 README.md
结构要求（全英文，因为开源=全球受众）：
- **Header**：项目 Logo（如有）、项目名称、一句话描述、徽章（Build、License、Docker Pulls、GitHub Stars）
- **Demo**：1-2 张核心界面截图（ProjectBoard + AI Chat）
- **Features**：6 阶段工作流、AI 辅助、宠物陪伴、自托管友好
- **Tech Stack**：前端/后端/数据库/AI 图标列表
- **Quick Start（Docker）**：
  ```bash
  git clone https://github.com/yourname/sparkbin.git
  cd sparkbin
  cp .env.example .env
  docker compose up -d
  # 访问 http://localhost
  ```
- **Self-Hosting**：链接到 `SELF_HOSTING.md`
- **Screenshots**：3-5 张关键页面截图
- **Roadmap**：已完成的 + 计划中的（开源社区可以看到方向）
- **Contributing**：链接到 `CONTRIBUTING.md`
- **License**：说明使用的许可证及含义摘要

### 3. 更新/创建 CONTRIBUTING.md
- 开发环境搭建（前后端并行启动）
- 分支命名规范：`feat/`、`fix/`、`docs/`、`refactor/`
- Commit message 规范（参考现有提交历史风格）
- PR 流程：fork → branch → PR → CI 通过 → review
- 代码规范：命名导出、interface 优先、Result 模式
- 测试要求：E2E 测试覆盖关键路径
- 禁止事项：不写死 API Key、不修改 migrations

### 4. 创建 SECURITY.md
- 漏洞报告邮箱/方式（可先用 GitHub Security Advisories）
- 支持版本（当前 master 分支）
- 安全更新通知方式
- 已知安全实践（密码 bcrypt、SQL 参数化、XSS 防护等已实施的措施）

### 5. 创建 GitHub 模板
- `.github/ISSUE_TEMPLATE/bug_report.md`：Bug 报告模板
- `.github/ISSUE_TEMPLATE/feature_request.md`：功能请求模板
- `.github/PULL_REQUEST_TEMPLATE.md`：PR 描述模板

### 6. 创建/规范化 CHANGELOG.md
- 使用 [Keep a Changelog](https://keepachangelog.com/) 格式
- 从最近几次 commit 开始反向整理：
  - `security`：XSS、数据泄漏修复
  - `feat`：邮箱注册、OAuth、AI 宠物
  - `fix`：E2E 测试修复

### 7. 仓库清洁度检查
- 确保没有 `.env`、数据库文件、密钥提交到仓库
- 检查 `git ls-files | grep -E '\.env|\.db|secret'` 无输出
- 确保 `.gitignore` 已包含：`.env*`、`*.db`、`__pycache__/`、`node_modules/`
