## 技术实现亮点

### 1. 拖拽系统架构

```
DndContext (全局拖拽上下文)
├── SortableContext (排序上下文)
│   ├── SortableNote (可排序的便利贴)
│   └── useSortable (拖拽逻辑 hook)
├── sensors (传感器配置)
│   ├── PointerSensor (鼠标/触摸)
│   └── KeyboardSensor (键盘无障碍)
└── collisionDetection (碰撞检测)
```

### 2. 图片上传实现

```typescript
// Base64 编码存储
const reader = new FileReader();
reader.onloadend = () => {
  const base64 = reader.result as string;
  onChange(base64); // 存储到数据库
};
reader.readAsDataURL(file);
```

**优势**: 无需额外存储服务，图片直接存到数据库  
**限制**: 单张图片不能超过 2MB

### 3. 月视图算法

```typescript
const days = useMemo(() => {
  const monthStart = startOfMonth(currentDate);
  const monthEnd = endOfMonth(currentDate);
  const calendarStart = startOfWeek(monthStart, { locale: zhCN });
  const calendarEnd = endOfWeek(monthEnd, { locale: zhCN });

  return eachDayOfInterval({ start: calendarStart, end: calendarEnd });
}, [currentDate]);
```

使用 `date-fns` 处理日期计算，支持中文本地化。

## 遇到的问题与解决方案

### 问题1: Windows 文件编码问题

**现象**: JSX 文件中的模板字符串和特殊字符出现转义错误

**影响**: 
- ProjectDetail.tsx 修改困难
- 字符串替换时出现 `\` 和 `{` }` 转义问题

**解决方案**:
1. 使用 Write 工具直接写入完整文件
2. 避免使用 sed 进行复杂的字符串替换
3. 创建新组件文件而非修改大文件

### 问题2: ValidateStage 跨列拖拽复杂度

**现象**: 看板三列之间的拖拽需要重构大量代码

**影响**: 
- 需要重写 KanbanColumn 和 ValidationCard
- 需要处理跨列状态同步
- 测试覆盖需要大量用例

**决策**: 延后实现，当前使用按钮操作状态
