# analyses/ — 分析报告

> **本目录用途**：存放深度分析、战略思考、外部对标类文档。偏"战略层"，不直接指导代码编写，但指导产品方向和技术路线选择。

## 什么内容放这里？

| 内容类型 | 示例 |
|---------|------|
| 项目全景深度分析 | 技术架构评估、AI 适配度评分、国际化流程对照 |
| 创始人/负责人战略思考 | 优先级决策、资源分配建议、商业化路径分析 |
| 竞品分析 | 与 Notion、Linear、Trello 等工具的对比 |
| 用户调研报告 | 种子用户访谈总结、行为数据分析 |
| 技术债务评估 | 当前架构的瓶颈、未来 6 个月的技术路线图 |

## 命名规范

`YYYY-MM-DD-{主题}.md`

例如：
- `2026-04-27-项目全景深度分析报告.md`
- `2026-04-27-创始人战略关注清单.md`

## 与其他目录的区别

| 对比 | analyses/ | decisions/ | tasks/ |
|------|-----------|------------|--------|
| **性质** | 分析、思考、评估 | 决策、方向、战略 | 执行、代码、操作 |
| **产出物** | 报告、评分、建议 | 决策记录、方案对比 | 代码变更、测试结果 |
| **阅读时机** | 制定战略前 | 需要做重大选择时 | 执行具体开发时 |
