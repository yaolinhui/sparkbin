# SparkBin 移动端适配需求文档

## 一、项目概述

**目标**：为 SparkBin 的 Landing（营销站）和 Frontend（主应用）两个前端项目提供完整的移动端适配，确保在 375px-768px 宽度的设备上可用、易用。

**技术栈**：React + TypeScript + Tailwind CSS + Next.js（Landing）/ Vite（Frontend）

**设计原则**：
- 移动优先（Mobile First）的响应式策略
- 触摸目标最小 44px
- 内容优先级重排：核心功能优先显示
- 保持 Brutalist 设计风格的一致性

## 三、Frontend 项目适配需求

### 3.1 ProjectBoard.tsx（项目看板）

**问题**：
- 头部区域过于拥挤：系统状态、AI额度、同步时间、操作按钮全部挤在一行
- 无移动端专用导航

**修改方案**：
1. 头部左侧保留 Logo，右侧改为汉堡菜单按钮
2. 系统状态、AI额度、同步时间在下方的 Metrics Bar 中已有展示，头部可隐藏
3. 操作按钮（主题切换、语言切换、管理、账户、退出）收入移动端抽屉菜单
4. 统计卡片已经是 `grid-cols-2 md:grid-cols-4`，无需改动

### 3.2 ProjectDetail.tsx（项目详情）—— 最高优先级

**问题**：
- 左右分栏布局（编辑器 + AI聊天）在移动端完全不可用
- StageFlow 阶段流程条水平排列，可能溢出
- 头部按钮过多

**修改方案**：
1. **主布局**：桌面端保持 `flex-row`，移动端（< md）改为 `flex-col`
2. **AI 聊天面板**：
   - 移动端默认完全折叠（隐藏）
   - 添加浮动按钮在右下角展开 AI 聊天
   - 展开时以全屏覆盖层（overlay）形式呈现
3. **StageFlow**：
   - 在移动端改为可横向滚动的容器（`overflow-x-auto`）
   - 阶段按钮缩小字号和间距
   - 隐藏进度条和完成按钮的文本标签，仅显示图标
4. **头部按钮**：
   - 蓝图、Agent 按钮在移动端仅显示图标（已有 `hidden sm:inline`）
   - 更多菜单保持不变

### 3.3 AIChat.tsx（AI 聊天）

**问题**：
- 侧边栏模式固定宽度，在移动端会挤压编辑器空间
- 全屏模式的上下文面板固定宽度 `w-80`

**修改方案**：
1. 侧边栏模式在移动端的宽度问题通过 ProjectDetail 的布局改造解决（改为浮动全屏）
2. 全屏模式下，上下文面板在移动端改为可折叠的抽屉式
3. 输入框和按钮确保触摸目标 >= 44px

### 3.4 AdminPage.tsx（管理后台）

**问题**：
- 侧边栏固定宽度 `w-64`，在移动端占据大部分屏幕
- 整体布局是 `flex` 行排列
- 日志表格在小屏幕上会水平溢出

**修改方案**：
1. **侧边栏**：
   - 移动端改为顶部 Tab 切换（类似移动端常见的 Tab 导航）
   - 或改为可滑出的抽屉式菜单
2. **主内容区**：
   - AI 配置页面的两列布局在移动端改为单列
   - 提供商选择网格在移动端改为 2 列
3. **日志表格**：
   - 添加 `overflow-x-auto` 容器
   - 或改为卡片式布局（每张卡片一条日志）

### 3.5 LoginModal.tsx（登录弹窗）

**问题**：
- 弹窗内容在极小屏幕上可能溢出
- 触摸目标未显式优化

**修改方案**：
1. 弹窗最大宽度已经是 `max-w-md`，在移动端会自然收缩
2. 确保所有输入框和按钮高度 >= 44px（当前已经是 py-3，约 48px）
3. 无需大幅修改

### 3.6 LandingPage.tsx（未登录落地页）

**问题**：
- 3列网格在 320px 屏幕上会非常拥挤

**修改方案**：
1. 阶段网格改为 `grid-cols-2 sm:grid-cols-3`

### 3.7 ProfilePage.tsx（个人资料）

**问题**：
- InfoRow 使用 `flex justify-between`，在小屏幕上文字可能重叠
- OAuth 绑定区域在小屏幕上可能溢出

**修改方案**：
1. InfoRow 在移动端改为垂直布局（标签在上，值在下）
2. OAuth 绑定项在移动端改为垂直布局

### 3.8 StageFlow.tsx（阶段流程条）

**问题**：
- 所有阶段按钮和进度信息挤在一行
- 在小屏幕上必然溢出

**修改方案**：
1. 整个流程条容器添加 `overflow-x-auto`
2. 在移动端隐藏阶段名称文本，仅显示编号
3. 进度条和完成按钮在移动端简化显示

### 3.9 全局样式（index.css）

**修改方案**：
1. 添加 `-webkit-tap-highlight-color: transparent` 消除移动端点击高亮
2. 添加 `touch-action: manipulation` 优化触摸响应
3. 确保滚动条在移动端不占用过多空间（已有 8px，可接受）

## 五、验收标准

| 场景 | 标准 |
|------|------|
| iPhone SE (375px) | 所有页面无水平滚动条，核心功能可用 |
| iPhone 14 (390px) | 布局合理，触摸目标可点击 |
| iPad Mini (768px) | 与桌面端类似体验，布局自适应 |
| 所有设备 | 无内容截断、无按钮重叠、无文本溢出 |
