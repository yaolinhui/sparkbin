# decisions/ — 专项决策

> **本目录用途**：存放**跨领域重大决策**，通常涉及多个模块或具有长期影响（3~6 个月以上）。这些是项目方向的"转折点"。

## 什么内容放这里？

| 内容类型 | 示例 |
|---------|------|
| 商业模式决策 | 开源 vs 闭源、免费 vs 付费边界、定价策略 |
| 技术路线决策 | 数据库迁移（SQLite → PostgreSQL）、前端框架升级 |
| 品牌与合规决策 | 开源协议选择（MIT vs GPL）、数据隐私合规 |
| 组织决策 | 团队协作规范、代码审查流程、发布节奏 |

## 什么内容**不放**这里？

- 纯技术实现细节（如"为什么用 SSE 不用 WebSocket"）→ 放 `docs/01-架构决策记录.md`
- 单次开发任务的执行记录 → 放 `docs/tasks/`
- 分析报告（如竞品对比）→ 放 `docs/analyses/`

## 命名规范

`YYYY-MM-DD-{决策主题}.md`

例如：
- `2026-04-27-开源战略与商业化模式决策.md`

## 文档格式建议

每份决策文档至少包含：
1. **背景**：为什么要做这个决策？
2. **选项对比**：列出所有可行方案，含优缺点
3. **最终决策**：选中哪个方案，为什么？
4. **影响范围**：哪些模块、哪些人会受影响？
5. **回退方案**：如果决策错误，怎么撤销？
