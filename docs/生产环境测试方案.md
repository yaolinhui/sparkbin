# SparkBin 生产环境测试方案

> 适用场景：阿里云轻量服务器（后端）+ Vercel（前端）已上线后的持续验证
> 目标：低成本、自动化、不影响真实用户

---

## 一、测试分层架构

```
每次 Push
    └── CI/CD 部署
            └── 部署后自动验证（GitHub Actions）
                    ├── 本地冒烟测试（scripts/smoke-test-production.sh）
                    ├── 端到端测试（Playwright → 生产环境）
                    └── 性能基线检查（curl 计时）

每日定时
    └── 健康检查（Uptime Kuma / 阿里云监控）
            ├── 前端可访问性
            ├── 后端 API 响应
            └── SSL 证书有效期

每周手动
    └── 核心功能走查
            ├── 注册 → 创建项目 → AI 对话 → 提交阶段
            ├── 支付流程（Stripe Test Mode）
            └── 管理后台数据查看
```

## 二、部署后自动验证（每次 Push 后）

### 2.1 当前 CI/CD 的增强点

你的 `.github/workflows/deploy.yml` 目前只有：
```bash
curl -sf http://localhost:8000/health
```

**建议增强为验证生产域名**：

```yaml
# 在 deploy.yml 的 Step 5 之后追加：
- name: Production Smoke Test
  run: |
    sleep 10  # 等待服务完全启动
    bash scripts/smoke-test-production.sh
  env:
    TEST_USERNAME: ${{ secrets.TEST_USERNAME }}
    TEST_PASSWORD: ${{ secrets.TEST_PASSWORD }}
```

### 2.2 冒烟测试脚本

已创建 `scripts/smoke-test-production.sh`，覆盖：

| 检查项 | 通过标准 | 失败影响 |
|--------|---------|---------|
| 前端可访问 | HTTP 200 | 用户无法打开应用 |
| Landing 可访问 | HTTP 200 | SEO/引流中断 |
| 后端 /health | HTTP 200 | 所有 API 不可用 |
| /auth/me 未授权 | HTTP 401/403 | 认证中间件失效 |
| SSL 证书 | 剩余 > 30 天 | 浏览器安全警告 |
| 响应时间 | < 0.5s | 用户体验差 |
| 登录接口 | 返回 token | 用户无法登录 |

**运行方式**：
```bash
# 本地运行（需配置测试账号）
export TEST_USERNAME=your_test_user
export TEST_PASSWORD=your_test_pass
bash scripts/smoke-test-production.sh
```

## 三、端到端测试（Playwright → 生产环境）

### 3.1 为什么需要生产 E2E？

- 本地测试通过不代表生产可用（环境变量、CORS、SSL、CDN 都可能不同）
- 上线后代码与配置可能脱节（如域名、API Key、Webhook）

### 3.2 配置

已创建 `frontend/playwright.config.production.ts`，指向 `https://sparkbin.wanchun.me`。

**运行方式**：
```bash
cd frontend
npx playwright test --config=playwright.config.production.ts
```

**推荐测试范围**：
- `auth.setup.ts` → 登录并保存状态
- `project-board.spec.ts` → 项目列表加载
- `project-detail.spec.ts` → 项目详情导航
- **不要运行**：会创建/删除大量数据的测试（避免污染生产数据库）

### 3.3 生产 E2E 注意事项

1. **专用测试账号**：在数据库中创建一个 `test_user`，避免使用真实用户数据
2. **测试数据隔离**：测试创建的项目标题加前缀 `[TEST]`，便于后续清理
3. **运行时机**：避开用户高峰期（如北京时间 9:00-12:00, 14:00-18:00）
4. **频率**：每次部署后运行一次，或每日凌晨定时运行

### 4.3 阿里云服务器监控

阿里云轻量服务器控制台提供：
- CPU / 内存 / 带宽使用率（免费）
- 磁盘 I/O 监控
- 告警规则设置（如 CPU > 80% 发邮件）

**建议配置**：
1. 登录阿里云控制台 → 轻量应用服务器 → 监控
2. 设置告警规则：
   - CPU 使用率 > 80% 持续 5 分钟 → 发送邮件
   - 内存使用率 > 85% 持续 5 分钟 → 发送邮件
   - 带宽出站 > 80% 持续 10 分钟 → 发送邮件

## 五、安全测试

### 5.1 SSL 证书监控

冒烟测试脚本已包含证书有效期检查。额外建议：
- 设置日历提醒：证书到期前 14 天手动续期
- 或使用 Certbot 自动续期（已配置 fix-ssl.sh）

### 5.2 敏感端点测试

手动验证以下端点**未授权访问应被拒绝**：

```bash
# 应返回 401/403
curl -s -o /dev/null -w "%{http_code}" https://api-sparkbin.wanchun.me/admin/logs
curl -s -o /dev/null -w "%{http_code}" https://api-sparkbin.wanchun.me/ai/configs
curl -s -o /dev/null -w "%{http_code}" https://api-sparkbin.wanchun.me/payments/subscription-status

# 应返回 404 或 405（不允许直接访问）
curl -s -o /dev/null -w "%{http_code}" -X POST https://api-sparkbin.wanchun.me/payments/webhook
```

### 5.3 CORS 配置验证

```bash
# 从非白名单域名发起请求，应被 CORS 拒绝
curl -s -o /dev/null -w "%{http_code}" \
  -H "Origin: https://evil.com" \
  https://api-sparkbin.wanchun.me/health
```

## 六、持续监控（7×24）

### 6.1 推荐方案：Uptime Kuma（免费自托管）

在阿里云服务器上用 Docker 部署 Uptime Kuma：

```bash
docker run -d --restart=always -p 3001:3001 -v uptime-kuma:/app/data louislam/uptime-kuma:1
```

配置监控项：

| 名称 | URL | 检查间隔 | 预期状态 |
|------|-----|---------|---------|
| SparkBin 前端 | https://sparkbin.wanchun.me | 60s | 200 |
| SparkBin Landing | https://sparkbin.wanchun.me | 60s | 200 |
| SparkBin API | https://api-sparkbin.wanchun.me/health | 60s | 200 |
| SSL 证书 | api-sparkbin.wanchun.me:443 | 1h | 剩余 > 14 天 |

**告警通道**：
- 钉钉/飞书 Webhook（中国大陆推荐）
- Telegram Bot（国际推荐）
- 邮件

### 6.2 后端日志监控

当前后端使用 Python `logging` 输出到 stdout。在 Docker 中可通过：

```bash
# 查看实时日志
docker logs -f sparkbin-backend --tail 100

# 或配置 logrotate 防止日志占满磁盘
docker exec sparkbin-backend sh -c "ls -lh /var/log/"
```

**建议**：配置阿里云日志服务（SLS）或自建 Loki 收集 Docker 日志。

### 6.3 数据库监控

```bash
# 进入 PostgreSQL 容器
docker exec -it sparkbin-postgres psql -U sparkbin -d sparkbin

# 检查关键指标
SELECT count(*) FROM users;
SELECT count(*) FROM projects WHERE status = 'active';
SELECT count(*) FROM ai_call_logs WHERE created_at > NOW() - INTERVAL '1 day';
```

## 八、紧急回滚预案

如果生产测试发现严重问题：

```bash
# 1. 在阿里云服务器上快速回滚到上一个镜像
cd /opt/sparkbin
git reset --hard HEAD~1
docker build -f docker/Dockerfile.backend -t sparkbin-backend:latest backend/
docker compose -f docker-compose.deploy.yml up -d

# 2. 或回滚到指定 commit
git reset --hard <上一个稳定commit>
# 然后重新构建
```
