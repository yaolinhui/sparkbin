# 任务记录：ASCII 宠物系统概念验证

**日期**：2026-04-27
**状态**：方案阶段，待用户确认后执行
**目标**：用 ASCII 艺术替代现有 Emoji 宠物，验证视觉风格可行性

## 背景

当前宠物系统使用系统 Emoji（🐱🤖🐼 等），与 Brutalist 设计风格产生割裂感。用户要求以 ASCII / 字符画宠物替代，参考 Claude Code 的 `Clawd` 和 `Claude Buddy` 终端宠物系统。

## 技术规格

| 属性 | 设定 |
|------|------|
| 画布尺寸 | 5 行高 × 16 字符宽 |
| 帧率 | 500ms / 帧（idle 呼吸循环） |
| 字符集 | 纯 ASCII（`0x20-0x7E`），禁止使用 Unicode 方块字符 |
| 字体 | JetBrains Mono，`line-height: 1`，`white-space: pre` |
| 颜色 | CSS `color` 绑定宠物主题色，单色无渐变 |
| 动画方式 | React state + `setInterval`，帧数组轮询 |

## 实施步骤

### Step 1：数据层 — 定义 ASCII 帧格式

**新建文件**：`frontend/src/components/ASCIIPet.frames.ts`

每种宠物定义为一个对象：
```typescript
export interface ASCIIPetFrames {
  id: string;
  name: string;
  color: string;
  idle: string[][];      // 循环播放
  blink: string[][];     // 随机触发，播放一次后回 idle
  happy: string[][];     // 点击/互动时播放一次
}
```

**首批宠物（2 种，验证模板）**：
- `cat` — 猫（温暖陪伴基调）
- `robot` — 机器人（科技感，测试非生物形态在 ASCII 下的表现）

### Step 2：组件层 — ASCIIPet 渲染组件

**新建文件**：`frontend/src/components/ASCIIPet.tsx`

Props：
```typescript
interface ASCIIPetProps {
  petId: string;
  size?: 'sm' | 'md' | 'lg';  // sm: AIChat 头像, md: 看板按钮, lg: 配置面板预览
  animation?: 'idle' | 'blink' | 'happy' | 'sleep';
  onClick?: () => void;
}
```

职责：
- 根据 `petId` 查找对应帧数据
- 根据 `size` 应用不同 CSS 缩放（`font-size` + `transform: scale`）
- 内部维护 `currentFrameIndex`，`setInterval(500ms)` 循环 `idle` 帧
- 支持外部通过 `animation` prop 触发一次性动画（blink/happy），播放完毕后自动回 idle

### Step 3：替换现有宠物显示点

**修改文件 1**：`frontend/src/components/ProjectBoard.tsx`

**修改文件 2**：`frontend/src/components/AIChat.tsx`

**修改文件 3**：`frontend/src/components/AIPetConfig.tsx`

### Step 4：兼容性处理

## 帧设计草案

### Cat（猫）— 温暖陪伴

```
Idle Frame 1:
    /\_/\
   ( o.o )
   (  >  )
    |   |

Idle Frame 2（呼吸，身体微缩）:
    /\_/\
   ( o.o )
    ( > )
     | |

Blink（眨眼）:
    /\_/\
   ( -.- )
   (  >  )
    |   |

Happy（被点击）:
    /\_/\
   ( ^o^ )
    \ > /
     | |
```

### Robot（机器人）— 科技伙伴

```
Idle Frame 1:
     [o_o]
    /|   |\
     | = |
     d   b

Idle Frame 2（天线闪烁）:
     +[o_o]
    /|   |\
     | = |
     d   b

Happy:
     [o_o]
    /|   |\
     \ = /
      | |
```

## 测试方案

1. **视觉一致性测试**：在 Windows / macOS / Linux 下检查 ASCII 渲染是否一致（重点确认 `line-height: 1` 无错位）
2. **动画流畅度测试**：idle 循环 500ms 切换是否自然，无闪烁感
3. **交互响应测试**：点击宠物触发 happy 动画，动画结束后是否正确回到 idle
4. **尺寸适配测试**：sm/md/lg 三种尺寸在各自容器内不溢出
5. **构建测试**：`npm run build` 无 TypeScript 错误
