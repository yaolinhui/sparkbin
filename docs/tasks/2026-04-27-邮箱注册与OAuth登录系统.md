# 任务：邮箱注册与 OAuth 登录系统

> **日期**：2026-04-27
> **分支**：`master`（直接提交）
> **提交**：`ec6e7a5`

---

## 背景

原有认证系统仅支持单一管理员账号（admin/admin），无注册功能，无法满足多用户使用需求。本次实现完整的多用户认证体系，支持邮箱注册（带邮件验证）和第三方 OAuth 登录（Google / GitHub）。

---

## 技术方案

### 后端

1. **User 模型扩展**（`backend/app/models.py`）
   - 新增字段：`email`, `email_verified`, `oauth_provider`, `oauth_id`, `avatar_url`
   - `username` 和 `password_hash` 改为 `nullable=True`（OAuth 用户可能无密码）

2. **邮件服务**（`backend/app/email.py`，新增）
   - 集成 Resend API
   - 开发环境无 `RESEND_API_KEY` 时自动 mock（打印到控制台）
   - 支持发送验证邮件和密码重置邮件

3. **认证路由扩展**（`backend/app/routers/auth.py`）
   - `POST /register`：邮箱注册，发送验证邮件
   - `GET /verify-email`：点击邮件链接验证邮箱
   - `POST /forgot-password`：发送密码重置邮件
   - `POST /reset-password`：使用 token 重置密码
   - `GET /oauth/google` / `GET /oauth/google/callback`：Google OAuth 授权码流程
   - `GET /oauth/github` / `GET /oauth/github/callback`：GitHub OAuth 授权码流程
   - OAuth 回调成功后重定向到前端，带 `?oauth_success=1&access_token=xxx&refresh_token=xxx`

4. **Token 体系扩展**（`backend/app/auth.py`）
   - 新增 `create_email_verification_token()`：24 小时有效期
   - 新增 `create_password_reset_token()`：24 小时有效期

5. **配置扩展**（`backend/app/config.py`）
   - 新增：`resend_api_key`, `resend_from_email`
   - 新增：`google_client_id`, `google_client_secret`
   - 新增：`github_client_id`, `github_client_secret`
   - 新增：`frontend_url`

6. **Schema 扩展**（`backend/app/schemas.py`）
   - 新增：`RegisterRequest`, `ForgotPasswordRequest`, `ResetPasswordRequest`, `VerifyEmailResponse`

7. **数据库迁移**
   - `backend/alembic/versions/ec65b6f922b7_add_user_email_and_oauth_fields.py`
   - SQLite 兼容：使用 `batch_alter_table` 处理 `nullable` 变更
   - 包含列存在性检查，支持幂等重跑

### 前端

1. **LoginModal 重构**（`frontend/src/components/LoginModal.tsx`）
   - 三标签页：登录 / 注册 / 找回密码
   - 登录表单保留用户名/密码
   - 注册表单：用户名 / 邮箱 / 密码 / 确认密码
   - 找回密码表单：邮箱输入
   - 新增 Google / GitHub OAuth 登录按钮（位于登录标签页下方）

2. **新增页面**
   - `frontend/src/pages/VerifyEmailPage.tsx`：处理邮件验证链接，调用 `authApi.verifyEmail(token)`
   - `frontend/src/pages/ResetPasswordPage.tsx`：密码重置表单（新密码 + 确认密码）

3. **App.tsx 路由更新**
   - 始终使用 `BrowserRouter`
   - 新增公开路由：`/verify-email`、`/reset-password`
   - 新增 `OAuthHandler` 组件：解析 `?oauth_success=1&access_token=...` 并自动登录

4. **API 层扩展**（`frontend/src/services/api.ts`）
   - 新增接口：`RegisterRequest`, `ForgotPasswordRequest`, `ResetPasswordRequest`, `VerifyEmailResponse`
   - 新增方法：`authApi.register()`, `authApi.forgotPassword()`, `authApi.resetPassword()`, `authApi.verifyEmail()`, `authApi.getOAuthUrl()`

5. **E2E 测试修复**
   - `frontend/e2e/auth.setup.ts` 和 `frontend/e2e/auth.spec.ts`
   - 登录按钮选择器从 `hasText: /登录|Login|Sign/i` 改为 `getByRole('button', { name: 'LOGIN' })`
   - 原因：新 LoginModal 的"登录"标签按钮会匹配旧正则

## 已知限制

1. **OAuth 配置需手动申请**：Google 和 GitHub 的 Client ID / Secret 需在对应开发者平台手动创建应用并配置回调 URL。
2. **Resend 生产环境需配置 API Key**：开发环境自动 mock，生产部署时需配置 `RESEND_API_KEY` 和 `RESEND_FROM_EMAIL`。
3. **OAuth 头像同步**：当前仅首次登录时抓取头像 URL，后续 OAuth 账号头像变更不会自动同步。
4. **邮箱验证非强制登录**：当前设计允许未验证邮箱的用户登录（后续可根据需求改为强制验证）。
