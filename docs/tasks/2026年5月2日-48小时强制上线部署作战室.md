## 二、部署架构

```
用户浏览器
    │
    ├─→ Vercel（前端 App）  https://sparkbin-app.vercel.app
    │   └── React + Vite SPA
    │
    ├─→ Vercel（Landing 页） https://sparkbin-landing.vercel.app
    │   └── Next.js 静态导出
    │
    └─→ Render（后端 API）   https://sparkbin-api.onrender.com
        └── FastAPI + PostgreSQL
```

> **中国大陆开发注意事项**：
> - Vercel / Render 均为海外服务，国内开发者访问 Dashboard 需要科学上网
> - 目标用户定位为 **海外 indie hacker**，服务器放海外是正确选择
> - 若未来需要服务国内用户，前端可额外部署到 Cloudflare Pages 或阿里云 OSS
> - Google OAuth / GitHub OAuth 在国内部分网络环境下受限，属于已知问题

## 四、当前阻塞项与风险

### 🔴 阻塞项（必须解决）

1. **邮箱验证码代码混入主干**
   - 当前工作区有 11 个文件的未提交修改，包含半成品邮箱验证码功能
   - 这些代码如果直接部署，注册流程可能报错
   - **方案**：创建独立上线分支，排除邮箱验证码修改；或把邮箱验证码入口隐藏

2. **生产环境变量未配置**
   - SECRET_KEY / ENCRYPTION_KEY 已生成，但尚未填入 Render
   - 前端 API URL 仍是占位符

### 🟡 风险项（需要关注）

3. **中国大陆访问速度**
   - Vercel / Render 在国内无节点，国内用户访问可能慢
   - **缓解**：目标用户为海外 indie hacker，MVP 阶段可接受

4. **AI API 调用稳定性**
   - DeepSeek / Kimi / OpenAI 调用依赖海外网络
   - **缓解**：后端放在 Render（海外），调用 AI API 本身就在海外网络环境

5. **Resend 邮件服务**
   - 需要验证域名才能发邮件
   - **缓解**：邮箱验证码本次不上线，其他邮件功能（如密码重置）暂时关闭

6. **Google OAuth 国内可用性**
   - 国内用户无法使用 Google 登录
   - **缓解**：保留用户名密码注册，OAuth 作为可选
