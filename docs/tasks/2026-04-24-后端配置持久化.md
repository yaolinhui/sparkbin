## 项目概述
SparkBin 是一个面向 Vibe 开发者的项目创意管理工具，前后端分离架构。
- 前端：React 18 + Vite + TypeScript + Tailwind + Zustand
- 后端：FastAPI + SQLAlchemy + SQLite（开发）/ PostgreSQL（生产）
- AI 代理模式：前端不接触 API Key，全部走后端代理

## 本次任务目标
将"仅存于前端 localStorage 的配置数据"迁移到后端数据库，实现真正的跨设备持久化。

## 具体工作内容

### 1. AI 宠物配置持久化（最高优先级）
当前问题：AIPetConfig.tsx 中的宠物类型、名字、性格、话痨程度全部存在 localStorage（key: sparkbin-pet-config），换浏览器/清缓存就丢失。

需要做的：
- 在 backend/app/models.py 的 User 模型中新增字段 `pet_config`（JSON 类型，存储宠物配置对象）
- 在 backend/app/schemas.py 中新增/更新对应的 Pydantic 模型
- 在 backend/app/routers/auth.py 中：
  - GET /me 返回用户信息时包含 pet_config
  - 新增 PUT /me/pet-config 接口用于更新宠物配置
- 确保数据库 migration 自动兼容（参考 main.py 中的 SQLite 兼容性补丁逻辑，动态添加列）
- 不要修改 alembic/versions/ 下的已有迁移文件

### 2. 用户偏好模型持久化检查（中等优先级）
当前状态：User 模型已有 preferred_model 字段，但需要检查 API 是否完整。

需要做的：
- 检查 GET /auth/preferred-model 和 PUT /auth/preferred-model 是否正常工作
- 检查前端是否真正在调用这些 API（而不是只存在 localStorage）
- 如有问题，修复后端接口或前端调用逻辑

### 3. 主题偏好持久化（可选，时间够就做）
当前问题：主题偏好（sparkbin-theme）仅存 localStorage。

需要做的：
- 在 User 模型新增 `theme_preference` 字段（String，'light' 或 'dark'）
- 新增对应的更新/获取 API
- 前端在登录后从后端读取主题偏好，覆盖 localStorage
