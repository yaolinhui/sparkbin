# tasks/ — 任务记录

> **本目录用途**：存放**每一次具体开发/设计/配置任务的完整执行记录**。这是项目的"操作日志"，也是 AI 学习历史、避免重复踩坑的核心素材。

## 什么内容放这里？

| 内容类型 | 示例 |
|---------|------|
| 功能开发 | 实现 OAuth 登录、添加 AI fallback、修复拖拽裁剪 |
| 架构调整 | 数据库迁移、API 重构、依赖升级 |
| 配置变更 | CI/CD 搭建、部署脚本更新、环境变量调整 |
| 测试相关 | E2E 测试补充、性能测试、安全审计 |
| 文档治理 | 文档体系重构、命名规范化、索引建立 |

## 命名规范

`YYYY-MM-DD-{任务简述}.md`

例如：
- `2026-04-27-邮箱注册与OAuth登录系统.md`
- `2026-04-24-前端核心交互打磨.md`

## 文档格式建议

每份任务记录至少包含：
1. **任务信息**：目标、涉及模块、预计耗时
2. **约束条件**：允许改哪些文件、禁止改哪些
3. **执行过程**：关键步骤、遇到的问题、临时决策
4. **代码变更**：改了哪些文件、核心逻辑说明
5. **验收结果**：测试是否通过、是否达到预期
6. **复盘**：哪些约束有效、哪些应该调整、Prompt 优化建议

## 与 `docs/03-任务边界约束.md` 的关系

- `03-任务边界约束.md` 是**模板**，用于任务开始前的规划
- `tasks/` 下的文件是**实例**，记录任务实际执行的全过程
