# 任务记录：登录注册安全增强与表单体验优化

**日期**：2026-04-27
**状态**：已完成（代码与测试）
**关联决策**：DEC-020（全面安全加固）

---

## 背景

用户在测试登录页时发现两个体验与安全问题：
1. 注册接口缺乏机器人防护与限流，存在被脚本批量注册垃圾账号的风险
2. 前端表单校验缺失，错误只能等后端返回才暴露
3. 密码框 placeholder 过长不美观，违背 placeholder 设计初衷

## 实施内容

### 1. 后端限流逻辑扩展

**文件**：`backend/app/auth.py`

- 将 `_login_attempts` 扩展为 `_auth_attempts: dict[str, deque]`，支持 action 维度（`{ip}:{action}`）
- 新增通用函数 `check_rate_limit(request, action)` 和 `record_rate_limit_failure(request, action)`
- 保留旧函数 `check_login_rate_limit` / `record_login_failure` 作为兼容包装
- 注册与登录共享同一套阈值：5 次 / 5 分钟

### 2. 注册端点加固 + Honeypot

**文件**：`backend/app/routers/auth.py`、`backend/app/schemas.py`

- `RegisterRequest` 新增可选字段 `honeypot: Optional[str]`
- 注册路由入口处调用 `check_rate_limit(req, "注册")`
- 所有注册尝试（无论成功失败）均计入限流，防止脚本通过构造合法请求绕过
- 校验 `request.honeypot`：若非空字符串，返回 400 "请求异常，请重试"（不暴露是 honeypot 触发）
- 注册失败（用户名/邮箱已存在、密码复杂度不足）时记录限流失败

### 3. 前端表单校验

**文件**：`frontend/src/components/LoginModal.tsx`

- 新增 `validateEmail`（正则：`^[^\s@]+@[^\s@]+\.[^\s@]+$`）
- 新增 `validateUsername`（长度 3-50）
- `handleRegister` 提交前校验链：
  1. Honeypot 检查（静默丢弃，不暴露）
  2. 用户名长度 3-50
  3. 邮箱格式
  4. 密码长度 ≥8
  5. 密码与确认密码一致
- 错误提示即时反馈，无需等待后端往返

### 4. Honeypot 隐藏字段

**文件**：`frontend/src/components/LoginModal.tsx`

```tsx
<input
  type="text"
  name="website"
  tabIndex={-1}
  autoComplete="off"
  className="absolute opacity-0 top-0 left-0 h-0 w-0 pointer-events-none"
  aria-hidden="true"
/>
```

- `name="website"`：常见 bot 会填写
- `tabIndex={-1}`：防止键盘焦点进入
- 视觉上完全隐藏但 DOM 中存在

### 5. Placeholder 美化 + 密码强度条

**文件**：`frontend/src/components/LoginModal.tsx`

- 注册密码 placeholder 从长规则文本改为 `"输入密码"`
- 输入框下方新增 helper text：`至少 8 位，含大写、小写、数字和特殊符号`
- 保留已有的实时密码强度条（5 项评分：长度/大写/小写/数字/特殊符号）
- 登录密码 placeholder 保持 `"Password"`（与 Username 对齐）

### 6. API 契约更新

**文件**：`frontend/src/services/api.ts`

- `RegisterRequest` 接口新增 `honeypot?: string`
- 修复登录/注册 401 错误文案：login/register 的 401 不再触发 refresh token 重试，直接展示后端错误消息
