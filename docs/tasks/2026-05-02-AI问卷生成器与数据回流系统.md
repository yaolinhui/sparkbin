# AI 问卷生成器与数据回流系统 — 完整实施计划

## 目标
将「验证工具箱-问卷」从「AI 文字生成」升级为「AI 工具生成」：
- AI 生成结构化问卷 JSON
- 一键发布为可访问的独立网页
- 支持 PC/移动端填写
- 数据回流至项目看板
- 支持 AI 综合分析

## 架构决策（已确认）
1. 问卷页：完全独立沉浸（无导航栏）
2. 用户编辑：AI 直接生成完整 JSON，预览页提供「重新生成」
3. AI 分析：手动触发 + 满 10 份自动触发
4. 自定义域名：MVP 不做

---

## Phase A（发布前必须完成）

### A1. 数据库层
- [ ] `surveys` 表：id, project_id, user_id, public_id, title, description, config(JSONB), status, response_count, created_at
- [ ] `survey_responses` 表：id, survey_id, project_id, answers(JSONB), respondent_meta(JSONB), created_at
- [ ] Alembic migration（SQLite 兼容 batch_alter_table）

### A2. 后端 API
- [ ] `POST /projects/{id}/surveys/generate` — AI 生成问卷配置
- [ ] `GET /projects/{id}/surveys` — 列出项目问卷
- [ ] `PATCH /projects/{id}/surveys/{survey_id}/publish` — 发布/关闭问卷
- [ ] `GET /surveys/{public_id}` — 公开获取问卷（无认证）
- [ ] `POST /surveys/{public_id}/responses` — 提交回答（无认证，IP 限流）
- [ ] `GET /projects/{id}/surveys/{survey_id}/responses` — 获取回答列表

### A3. 前端类型与服务
- [ ] `types/index.ts` 增加 Survey, SurveyResponse, SurveyConfig, Question 类型
- [ ] `services/api.ts` 增加 surveyApi 方法
- [ ] `services/ai.ts` 增加 `generateSurveyConfig` 方法（AI 生成 JSON）

### A4. 问卷公开页
- [ ] 新路由 `/s/:public_id`
- [ ] `SurveyPublicPage.tsx` — 独立沉浸页，Brutalist 设计
- [ ] `SurveyForm.tsx` — 表单渲染组件（单选/多选/评分/文本）
- [ ] 移动端触摸友好的大按钮选项
- [ ] 进度条显示
- [ ] 提交后感谢页

### A5. ValidateStage 改造
- [ ] 问卷工具生成流程：点击 → AI 生成 → 预览 → 发布
- [ ] 预览面板：显示问卷题目 + 发布按钮 + 重新生成按钮
- [ ] 发布后显示：公开链接、二维码、回收数量
- [ ] 工具卡片改造：survey 类型显示「查看问卷」而非「复制内容」

## 关键约束
- 单一代码库，自托管和 SaaS 共用
- AI 额度制已生效，生成问卷消耗 1 次 AI 调用
- 公开提交接口需 IP 限流（防刷）
- 所有新增代码遵循 Brutalist 设计系统
