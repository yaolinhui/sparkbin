## 二、中国大陆可行的部署方案对比

### 方案 A：Render（免费）+ Vercel（免费）—— 本周日首选

**后端：Render**
- Web Service：免费（15分钟休眠，冷启动 30-60 秒）
- PostgreSQL：免费（90 天有效期，到期需重建或导出数据）
- 节点：美国/欧洲
- 已有配置：`render.yaml` + `runtime.txt`

**前端：Vercel**
- 完全免费， generous 额度

**优点**：
- 项目已有完整配置，10 分钟可部署
- 零成本
- 本周日就能上线

**缺点**：
- Render Dashboard 需要科学上网
- 后端冷启动慢（首次访问要等）
- 免费 PostgreSQL 90 天后需迁移

**适合**：本周日紧急上线，验证产品

---

### 方案 B：Fly.io（免费）+ Vercel（免费）—— 亚洲节点优选

**后端：Fly.io**
- 免费额度：3个共享 CPU 实例 + 3GB 存储
- 节点：新加坡（亚洲，延迟低）
- PostgreSQL：可跑免费 Postgres

**优点**：
- 新加坡节点，国内开发者测试延迟低
- 免费额度对 MVP 足够
- 命令行部署，适合自动化

**缺点**：
- 需要学 `flyctl` 命令行工具
- 需要写 `fly.toml` 配置文件
- 文档不如 Render/Vercel 丰富

**适合**：愿意花 1-2 小时学习命令行，追求亚洲节点

---

### 方案 C：国内轻量服务器（~99元/年）+ Vercel（免费）—— 长期最稳

**后端：阿里云/腾讯云轻量应用服务器**
- 价格：新用户 99 元/年（2核2G3M带宽）
- 系统：Ubuntu / Debian
- 部署：Docker + Docker Compose
- 数据库：服务器本地 PostgreSQL 或 Docker 容器

**优点**：
- 一年成本仅 99 元，性价比极高
- 完全可控，无冷启动
- 一台服务器可以跑多个项目（每周新产品用子域名区分）
- 国内访问速度快

**缺点**：
- 需要手动配置服务器（Docker、Nginx、SSL）
- 服务器在国内，.me 域名 + 海外用户访问理论上无需备案，但存在灰色地带
- 需要买服务器（虽然便宜）

**适合**：建立"每周日发布产品"的长期标准化流程

---

### 方案 D：Oracle Cloud（永久免费）—— 薅羊毛之王

**后端：Oracle Cloud Infrastructure**
- 免费 tier：2个 AMD 实例 + 4个 ARM 实例 + 2个 Block Storage + 2个数据库
- 节点：可选日本/韩国/新加坡
- 有效期：永久（只要不超免费资源配额）

**优点**：
- 真·永久免费
- ARM 实例性能很强
- 可以跑 Docker Compose

**缺点**：
- 注册需要海外信用卡（部分国内双币卡可用）
- 注册审核可能需要几天
- 有回收风险（长期不用可能回收实例）
- 需要一定服务器运维知识

**适合**：有时间注册、愿意折腾基础设施

## 三、督战推荐：分阶段执行

### 第一阶段：本周日上线（方案 A - Render + Vercel）

**为什么先选 Render：**
1. 项目已有 `render.yaml`，零配置成本
2. 10 分钟就能部署好
3. 先让产品跑起来，验证价值
4. 本周日的目标不是"完美架构"，而是"能访问、能注册、能用"

**执行步骤**：
```
1. 注册 Render（需要科学上网）
2. Dashboard → Blueprints → New Blueprint → 选择 sparkbin 仓库
3. Render 自动读取 render.yaml 创建 Web Service + PostgreSQL
4. 修改环境变量（CORS 改为 https://app.wanchun.me）
5. 等待构建完成，拿到 api 域名
6. Vercel 部署前端（填入 Render API URL）
7. 绑定域名 app.wanchun.me / api.wanchun.me
```

### 第二阶段：下周迁移到长期方案（方案 C - 国内轻量服务器）

**为什么迁移：**
- Render 免费 PostgreSQL 90 天后过期
- 每周发布新产品，需要一个稳定、可控的"底座"
- 99 元/年的成本可以忽略不计

**执行步骤**：
```
1. 购买阿里云/腾讯云轻量服务器（新用户 99 元/年）
2. 服务器上安装 Docker + Docker Compose
3. 配置 Nginx 反向代理 + SSL（Let's Encrypt）
4. 写一套 docker-compose.yml 模板
5. 以后每周新产品：复制模板 → 改项目名 → 跑 docker-compose up
6. 子域名用 api-project1.wanchun.me, api-project2.wanchun.me
```

### 第三阶段：终极优化（可选）

- 如果 Oracle Cloud 注册成功，把长期项目迁过去（真·永久免费）
- 或者用 Cloudflare Tunnel 把家里的 NAS/服务器暴露出来（零成本）

## 六、Render 部署立即行动清单

如果你选 A，下面是精确步骤：

### Step 1：注册 Render（5分钟）
1. 科学上网打开 https://render.com
2. Sign Up with GitHub
3. 授权访问你的 sparkbin 仓库

### Step 2：一键部署（3分钟）
1. Dashboard → Blueprints → New Blueprint
2. 选择 `yaolinhui/sparkbin`（或 `yerendong/sparkbin`）
3. 分支选 `deploy/production`
4. Render 自动读取 `render.yaml`，创建：
   - `sparkbin-api`（Web Service，Python）
   - `sparkbin-web`（Static Site，前端）
   - `sparkbin-db`（PostgreSQL）

### Step 3：配置环境变量（5分钟）
1. 进入 `sparkbin-api` → Environment
2. 修改以下变量：
```
CORS_ORIGINS=https://app.wanchun.me
APP_URL=https://app.wanchun.me
FRONTEND_URL=https://app.wanchun.me
SECRET_KEY=PLrKNYTVPKqMvSHdesSIjMZLEXo46UdO6k05iefq-3H2Ps7_zL801noTyyz2uhI7
ENCRYPTION_KEY=86B-dqIThpDgrhNYHvjlXefzJ1QbXn5wL0lwP5KGsLA=
```
3. 进入 `sparkbin-web` → Environment
4. 修改：`VITE_API_URL=https://sparkbin-api.onrender.com`

### Step 4：等构建完成（5-10分钟）
- 两个服务都变绿后，访问 `https://sparkbin-api.onrender.com/health` 验证后端
- 访问 `https://sparkbin-web.onrender.com` 验证前端

### Step 5：绑定域名（10分钟）
1. `sparkbin-web` → Settings → Custom Domains → 添加 `app.wanchun.me`
2. 按提示去域名服务商添加 CNAME 记录
3. `sparkbin-api` → Settings → Custom Domains → 添加 `api.wanchun.me`
4. 更新 `sparkbin-web` 的 `VITE_API_URL=https://api.wanchun.me`
