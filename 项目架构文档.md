# SparkBin 项目架构文档

## 1. 项目目标

将纯前端应用改造为前后端分离架构，解决前端存储 GitHub Token 的安全隐患。

---

## 2. 架构总览

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│    前端 (Vite)   │────▶│  后端 (FastAPI) │────▶│  GitHub API     │
│   React + TS    │◄────│   Python 3.10+  │◄────│   JSON 存储     │
│   localhost:5173│     │   localhost:8000│     │   私有仓库       │
└─────────────────┘     └─────────────────┘     └─────────────────┘
        │                       │
        │                       │
        ▼                       ▼
┌─────────────────┐     ┌─────────────────┐
│  localStorage   │     │   .env (Token)  │
│  UI 状态缓存     │     │   安全存储密钥   │
└─────────────────┘     └─────────────────┘
```

---

## 3. 后端设计 (FastAPI)

### 3.1 目录结构

```
backend/
├── main.py              # FastAPI 应用入口
├── config.py            # 配置管理（读取 .env）
├── models/
│   └── schemas.py       # Pydantic 数据模型
├── services/
│   └── github.py        # GitHub API 封装
├── routers/
│   └── projects.py      # API 路由
├── requirements.txt     # Python 依赖
├── .env.example         # 环境变量示例
└── .gitignore          # Git 忽略规则
```

### 3.2 API 端点设计

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/health` | 健康检查 |
| GET | `/api/projects` | 获取所有项目 |
| POST | `/api/projects` | 保存所有项目 |
| GET | `/api/sync/status` | 获取同步状态 |

### 3.3 数据流向

```
1. 前端请求获取数据
   Frontend ──GET /api/projects──▶ Backend ──GET /repos/.../contents──▶ GitHub
   Frontend ◄──JSON 数据─────────── Backend ◄──Base64 内容────────────── GitHub

2. 前端保存数据
   Frontend ──POST /api/projects──▶ Backend ──PUT /repos/.../contents──▶ GitHub
   Frontend ◄──success───────────── Backend ◄──updated file────────────── GitHub
```

### 3.4 环境变量配置

```bash
# backend/.env
GITHUB_TOKEN=ghp_xxxxxxxxxxxx
GITHUB_OWNER=your_username
GITHUB_REPO=sparkbin-data
GITHUB_FILE_PATH=projects.json
API_PORT=8000
CORS_ORIGINS=http://localhost:5173
```

## 5. 部署方案

### 5.1 开发环境启动流程

```bash
# 1. 启动后端
cd backend
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
# 创建 .env 文件并配置
uvicorn main:app --reload --port 8000

# 2. 启动前端
cd ../sparkbin
npm run dev
```

### 5.2 生产部署方案（未来）

| 方案 | 优点 | 缺点 |
|------|------|------|
| 本地运行 | 简单、免费、数据私密 | 仅本机访问 |
| Vercel + Serverless | 免费托管、自动部署 | 需要适配 Serverless |
| 树莓派/NAS | 个人专用、永久在线 | 需要维护硬件 |

## 6. 安全考虑

1. **Token 存储**：GitHub Token 仅存储在后端 .env 文件，不暴露给前端
2. **CORS 限制**：后端只允许特定域名访问（开发时为 localhost）
3. **无认证**：当前仅本地使用，不添加用户认证（简化）
4. **数据备份**：GitHub 仓库天然具备版本控制和备份功能

## 8. 技术选型说明

| 选择 | 理由 |
|------|------|
| Python + FastAPI | 现代化、自动文档、异步支持 |
| 本地运行 | 个人使用场景最简单，无需服务器成本 |
| .env 配置 | 标准做法，易于管理和切换环境 |
| 代理模式 | 最小改动，保持前端逻辑基本不变 |
