# 📄 SparkBin 项目报告

## 1. 项目概述

| 项目 | 内容 |
|------|------|
| **名称** | SparkBin |
| **定位** | 个人创意/点子管理工具，深度规划看板 |
| **核心目标** | 明确每个点子所处阶段，可视化项目生命周期，提示"下一步该怎么做" |
| **用户** | 仅个人使用 |
| **数据存储** | 纯前端 + GitHub JSON 文件 |

---

## 2. 核心功能架构

### 2.1 两层状态系统

```
┌─────────────────────────────────────────────────────────┐
│  第一层：项目级状态（控制整体显示）                         │
│  ├── 🟡 暂停（灰色显示）                                  │
│  ├── 🟢 启用（正常显示）                                  │
│  └── ⚪ 归档（折叠到最后，只显示序号+标题）                 │
├─────────────────────────────────────────────────────────┤
│  第二层：阶段级状态（项目内部进度）                         │
│  想法 → 调研 → 开发 → 完成 → 上线 → 宣传                    │
│   ●      ○      ○      ○      ○      ○                  │
│  （当前位置用圆形高亮，横向流程图显示）                      │
└─────────────────────────────────────────────────────────┘
```

### 2.2 界面结构

**首页（项目面板）：**
- 平铺显示，带序号
- 缩放层级：10项目 → 5项目 → 1项目（滚轮缩放）
- 每个面板：图形图标 + 标题 + 痛点 + 当前阶段
- 右下角矩形"+"按钮添加项目

**项目详情（分屏界面）：**
- 左侧：富文本编辑器（Tiptap），可实时修改阶段内容
- 右侧：AI 对话框，AI 主动提问引导，用户回答后生成内容显示在左侧
- 顶部：横向流程图（React Flow），显示6个阶段，当前阶段高亮

## 3. 工作流程

### 3.1 创建新项目

```
1. 点击 "+" 按钮
2. 输入白话痛点描述
3. 调用 Kimi Code API 优化为精致描述
4. 用户确认/修改后，进入"想法"阶段
5. 项目出现在首页面板
```

### 3.2 阶段推进流程

```
当前阶段（启用状态）
    ↓
在左侧富文本编辑器记录内容
    ↓
右侧 AI 对话框辅助思考（可选）
    ↓
点击大大的 ✓ "完成此阶段"
    ↓
当前阶段锁定（只读），开启下一阶段新文档
    ↓
进入下一阶段编辑界面
```

### 3.3 各阶段特点

| 阶段 | 核心任务 | AI 辅助 |
|------|---------|---------|
| 想法 | 记录灵感、初步思考 | 引导理清思路 |
| 调研 | 市场分析、竞品调研、技术可行性 | 生成调研框架、整理笔记 |
| 开发 | 功能规划、技术选型、Bug 记录 | 代码建议、任务分解 |
| 完成 | 项目收尾、测试 | 检查清单 |
| 上线 | 部署、发布 | 上线检查清单 |
| 宣传 | 渠道选择、内容创作 | **自动生成宣传模板和渠道建议** |

## 6. 视觉设计规范

### 6.1 色彩方案
- **主色调**：极简黑白
- **背景**：纸张纹理（轻量化，不厚重）
- **暂停状态**：灰色
- **启用状态**：正常黑色文字
- **当前阶段**：圆形高亮标记

### 6.2 图形元素
- 多使用圆形、图标、标点符号
- 思维导图风格的连接线
- 避免"AI 味"过重的设计
- 矩形面板带轻微阴影

---

## 7. 同步机制

1. **自动保存**：编辑后自动 commit + push 到 GitHub
2. **刷新同步**：浏览器刷新后自动拉取最新 JSON
3. **冲突处理**：简单策略，以最新时间戳为准（单用户场景）

---

## 8. 项目边界（明确不做）

| 不做 | 原因 |
|------|------|
| 手机/平板适配（第一版） | 电脑端优先，后续迭代 |
| 分享/协作功能 | 仅限个人使用 |
| 数据导出（Markdown/图片） | 内部工具，功能精简 |
| 语音交互（第一版） | 比喻性描述，暂不实现 |
| 复杂权限系统 | 公开 GitHub 仓库，无需权限 |
| 提醒/通知功能 | 手动操作，无需监听 |
