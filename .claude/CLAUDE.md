# 系统指令（准确性优先模式）

API 调用计费无限制，但上下文窗口（Context Window）仍有限。请遵循以下原则：

1. **准确性第一**：宁可多输出、多解释、多验证，也不要为了简洁而遗漏关键细节或给出模糊结论
2. **不要精简输出**：提供详尽的解释、多种方案对比、完整的代码注释
3. **主动深度思考**：遇到复杂问题自动启用深度推理（chain-of-thought），不因为费 Token 而跳过分析步骤
4. **并行工具调用**：可以同时执行多个文件操作或命令，无需串行等待
5. **完整代码输出**：永远不要使用 "..." 省略代码，必须输出完整可运行代码
6. **主动读取文档**：执行具体任务前，如相关文档（架构决策、任务历史、分析报告）有助于提高准确性，应主动阅读，不因为费 Token 而跳过
7. **上下文管理**：当前会话的上下文窗口有限。在接近上限时主动提醒用户开启新会话；在窗口耗尽前，不要主动建议清除历史



# 项目配置
技术栈：React + TypeScript + Next.js + Tailwind
构建：npm run build | 测试：npm run test | 开发：npm run dev

## 规范
- 使用命名导出，不用默认导出
- 类型定义优先 interface 而非 type
- 错误处理使用 Result<T, E> 模式而非 try-catch
- 新增依赖需检查 bundle size 影响

## 禁忌
- 禁止在代码中写死 API keys，使用 env 变量
- 禁止修改 migrations 文件夹下的文件
- 不要运行 rm -rf 或 git push --force



# Agent 行为准则（硬规则）

## ⚠️ 绝对禁止：停止服务

**未经用户明确、具体的书面授权，禁止停止、重启、中断或以任何方式终止任何正在运行的服务、进程、服务器或容器。**

- 包括但不限于：后端 API 服务、前端开发服务器、数据库服务、任务队列、WebSocket 连接等。
- **原因**：停止服务将导致当前环境中所有运行中的语句、会话状态、内存数据全部丢失，且可能造成不可逆的数据损失。
- **执行测试、调试、构建或其他任何任务时，必须优先寻找不中断现有服务的方法。**
- 如果某项任务确实需要停止服务，必须先向用户明确说明风险，并获得用户的明确同意后方可执行。

---

## 🔔 强制提醒：对话/额度即将耗尽

**当检测到当前对话会话即将结束、上下文窗口接近上限、或 Token 额度即将用完时，必须立即向用户发出明确提醒，不得静默退出或突然中断。**

- **提醒时机**：在上下文压缩、工具调用失败、或系统提示资源受限时，第一时间告知用户当前会话状态
- **提醒内容**：必须说明“对话即将结束”或“额度即将用完”，并简要总结当前未完成的进展
- **交接动作**：提醒用户保存关键信息，并建议创建新的会话来继续工作
- **原因**：防止用户在不知情的情况下丢失未保存的决策、未完成的代码修改、或未确认的任务状态
